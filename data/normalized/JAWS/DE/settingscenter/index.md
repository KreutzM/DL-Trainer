# JAWS – Einstellungsverwaltung (Settings Center)

## Strukturierten Modus definieren

Verwenden Sie den Dialog Optionen für Elementtypen, um den strukturierten Modus so anzupassen, dass er die Informationen anzeigt, die Sie benötigen, und um die Darstellungsweise festzulegen. In diesem Dialogfeld gibt es drei Registerkarten: Elementeigenschaften, Statuseigenschaften und HTML-Attribute. Auf diesen Registerkarten legen Sie die Braille-Darstellung für die unterschiedlichen Elemente fest.

Um diesen Dialog zu öffnen, gehen Sie bitte folgendermaßen vor:

1. Öffnen Sie die Einstellungsverwaltung und geben Sie "Strukturierter Modus Erweitert" im Sucheingabefeld ohne Anführungsstriche ein.
2. Anschließend drücken Sie **PFEIL RUNTER **, um zu den gefilterten Ergebnissen der Strukturansicht zu springen.
3. Drücken Sie **LEERTASTE ** auf Erweitert, um das Dialogfenster Optionen für Elementtypen zu öffnen.

Sie verändern die Braille-Darstellung für ein Element wie folgt:

1. Mit **STRG+TAB ** öffnen Sie die jeweilige Registerkarte.
2. Drücken Sie **TAB ** und gehen Sie mit den Pfeiltasten auf das Steuerelement, den Status oder das HTML-Attribut, das Sie verändern möchten.
3. Drücken Sie **F2 **.
4. Geben Sie die neue Braille-Bezeichnung ein, die JAWS für das entsprechende Element verwenden soll, und drücken Sie danach **EINGABE **.
5. Aktivieren Sie den OK Schalter, um die vorgenommenen Änderungen zu speichern.

## Elementeigenschaften

Für jedes in der Liste der Elementtypen angezeigte Steuerelement gibt es verschiedene Informationen, die auf der Braillezeile angezeigt werden können. So wird zum Beispiel für einen Auswahlschalter der Elementname, der Elementstatus, der Elementtyp und die Positionsinformation angezeigt. Sie können festlegen, in welcher Reihenfolge diese Informationen angezeigt werden, und wie viel davon angezeigt wird - und das für jedes einzelne Steuerelement. Wählen Sie einen Elementtyp in der Liste aus und drücken dann **TAB **, um auf den Schalter Element ändern zu springen.

Ein Dialog wird geöffnet, der zwei Listenfelder und mehrere Schalter enthält. Das erste Listenfeld mit Namen Auf der Braillezeile Anzeigen enthält die Informationstypen, die für das von Ihnen gewählte Steuerelement angezeigt werden. Diese stehen in der Reihenfolge, in der sie auch auf der Braillezeile angezeigt werden. Die zweite Liste mit Namen Kann Hinzugefügt Werden enthält diejenigen Informationstypen, die zu dem Steuerelement gehören, momentan jedoch nicht auf der Braillezeile angezeigt werden. Wenn Sie einen Informationstypen aus der Liste Auf Braillezeile Anzeigen entfernen, wird er zur Liste Kann Hinzugefügt Werden hinzugefügt. Mit den Schaltern können Sie die Reihenfolge der Informationen in der Liste Auf Braillezeile Anzeigen verändern, und Sie können Einträge aus dieser Liste entfernen. Wenn Sie einen Eintrag aus der Liste Auf Braillezeile Anzeigen entfernen, wird dieser Informationstyp nicht mehr für dieses Steuerelement auf der Braillezeile angezeigt.

In der Liste Auf Braillezeile Anzeigen kann die Reihenfolge der Informationen, die auf der Braillezeile angezeigt werden, verändert werden, indem Sie **ALT+PFEIL RAUF ** oder ** ALT+PFEIL RUNTER ** drücken. Wenn ein bestimmter Informationstyp nicht angezeigt werden soll, wählen Sie ihn in der Liste aus, gehen Sie mit ** TAB ** zum Schalter Entfernen, und drücken Sie ** LEERTASTE **.

Um Informationen, die Sie entfernt haben, wieder anzuzeigen, gehen Sie mit **TAB ** zur Liste Kann Hinzugefügt Werden, wählen den Informationstyp, den Sie hinzufügen möchten, gehen mit ** TAB ** zum Schalter Hinzufügen, und drücken Sie die ** LEERTASTE **.

## Statuseigenschaften

Auf der Registerkarte Statuseigenschaften in diesem Dialog können Sie festlegen, wie der Status eines Steuerelements auf der Braillezeile angezeigt werden soll. Der Status eines Steuerelements kann lauten: Aktiviert, Nicht Aktiviert, Gewählt oder Nicht Gewählt.

## HTML Attribute

Auf der Registerkarte HTML Attribute legen Sie fest, wie JAWS bestimmte HTML Elemente in Braille darstellen soll. Sie können bestimmen, wie JAWS besuchte Links, anklickbare Elemente und Elemente mit dem MouseOver Attribut anzeigen soll.

## Braille Gruppe

## Allgemein

Nutzen Sie diese Optionen, um grundlegende Brailleeinstellungen und die Braillekurzschrift zu konfigurieren. Diese Optionen beinhalten Elemente für Einstellungen zur Kurzschriftübersetzung, zum Braillecursor und zum Weiterbewegen.

### Translation

Nutzen Sie die Einstellungen dieser Gruppe, um Brailleübersetzungseinstellungen zu konfigurieren

#### Sprache

Verwenden Sie dieses Eingabefeld, um die Braillesprache festzulegen. JAWS verfügt über die Möglichkeit, Braille in verschiedenen Sprachen anzuzeigen, darunter Englisch, Deutsch, Spanisch, Finnisch und Französisch, um nur ein paar zu nennen. Die Standardauswahl ist Deutsch - Deutschland.

#### Ausgabe

Nutzen Sie diese Ausklappliste, um den Ausgabemodus zu konfigurieren, also dem, was Sie in Braille auf Ihrer Braillezeile angezeigt bekommen. Die erste Option ist immer Computer Braille. Die anderen zur Verfügung stehenden Einstellungen hängen von der aktuell gewählten Sprache ab. Wenn beispielsweise die Sprache auf Deutsch - Deutschland gesetzt ist, dann sind die verfügbaren Ausgabemodi Deutsch Grade 0, Deutsch Grade1 und Deutsch Grade 2. Wenn Sie die Sprache auf US Englisch einstellen, dann stehen Ihnen Amerikanisches Englisch Grade 1, Amerikanisches Englisch Grade 2, Vereinheitlichtes Englisch Braille Grade 1 und Vereinheitlichtes Englisch Braille Grade 2 zur Verfügung.

#### Eingabe

Nutzen Sie diese Ausklappliste, um den Eingabemodus festzulegen, der verwendet wird, wenn Sie über die Perkins-Tastatur Ihrer Braillezeile in Braille schreiben. Die erste Option ist immer Computer Braille. Die anderen zur Verfügung stehenden Einstellungen hängen vom aktuell gewählten Ausgabemodus ab. Wenn Sie beispielsweise ausgewählt haben, dass Braille in Deutsch Kurzschrift angezeigt wird, dann können Sie für die Eingabe entweder Computerbraille oder Deutsch Kurzschrift wählen. Diese Einstellung ist nicht verfügbar, wenn der gewählte Ausgabemodus Computer Braille ist, oder wenn der gewählte Ausgabemodus keine Eingabe unterstützt.

#### Aktuelles Wort in Computerbraille anzeigen

Aktivieren Sie dieses Kontrollfeld, dann wird das Wort an der Cursorposition in Computerbraille angezeigt. Deaktivieren Sie dieses Kontrollfeld, um das Wort in Kurzschrift anzuzeigen. Dieses Kontrollfeld ist standardmäßig aktiviert. Das Kontrollfeld für Aktuelles Wort in Computerbraille anzeigen ist nur verfügbar, wenn der Ausgabemodus Grade 1 oder Grade 2 und der Eingabemodus Computerbraille ist.

#### Schriftschnittmarkierungen anzeigen

Aktivieren Sie dieses Kontrollfeld, um Braille-Indikatoren anzuzeigen, wenn Sie auf Text mit bestimmten Attributen treffen, wie etwa fett, unterstrichen oder kursiv. Dieses Kontrollfeld ist standardmäßig deaktiviert und nur verfügbar, wenn als Ausgabemodus Grade 1 oder Grade 2 verwendet wird.

#### Sprachprofile

Mehrsprachige Anwender können diese Option wählen, um das Dialogfenster für Sprachprofile zu öffnen, in welchem Sie Einstellungen für individuelle Brailleprofile konfigurieren können. Ein Brailleprofil besteht aus einem Übersetzungsmodus und den dazugehörigen Brailletabellen einer bestimmten Sprache für Eingabe und Ausgabe. In diesem Dialogfeld können Sie das primäre Brailleprofil festlegen und bevorzugte Profile auswählen. Sie können dann die Tastenkombination **STRG+UMSCHALT+WINDOWS+L ** verwenden, oder ** PUNKTE 2-3-4-5-7 CHORD ** auf einer Focus Braillezeile, um schnell zwischen bevorzugten Profilen zu wechseln, wenn Sie Text in anderen Sprachen lesen oder schreiben möchten.

Drücken Sie **LEERTASTE **, um das Dialogfenster zu öffnen.

### Braillemodus

In dieser Ausklappliste wird das Format festgelegt, in dem Informationen zu Ihrer Braillezeile gesendet werden. Die folgenden Optionen stehen zur Verfügung:

- **Fläche **: In diesem Modus wird die Zeile, auf der sich der Cursor befindet, zur Braillezeile gesendet.
- **Strukturiert **: In diesem Modus werden für die aktuelle Cursorposition relevante Informationen zur Braillezeile gesendet. Dies ist die Standardeinstellung.
- **Sprachausgabe **: In diesem Modus werden an die Braillezeile dieselben Informationen gesendet wie an die Sprachausgabe.

### Weiterbewegen

Nutzen Sie die Optionen dieser Gruppe, um einen Nachführungsmodus bei Nutzung einer Braillezeile zu definieren.

#### Benutzerdefiniertes Weiterbewegen

Das Weiterbewegen der Braillezeile bewegt im Allgemeinen die Zeile nach links oder nach rechts um die Länge der Zeile weiter. Sie tun dies mit dem Drücken der Taste zum Weiterbewegen auf Ihrer Braillezeile. Wählen Sie eine der folgenden Optionen, wie JAWS beim Weiterbewegen Ihre Braillezeile aktualisieren soll:

- **Am Passendsten ** - Wörter werden beim Weiterbewegen nicht abgeschnitten.
- **Fester Ausschnittswechsel ** - Stellt sicher, dass sich die Braillezeile stets die exakte Anzahl von Zellen, die auf der Braillezeile verfügbar sind, weiterbewegt.
- **Text maximieren ** - Es wird die maximale Textmenge, die auf die Braillezeile passt, angezeigt. JAWS zeigt "abgeschnittene" Wörter an beiden Enden der Zeile an, um mehr Informationen über den Zusammenhang zu liefern.
- **Automatisch ** - JAWS wählt die beste Methode zur Textanzeige auf Ihrer Braillezeile. JAWS verwendet die Methode Am Passendsten und schaltet auf Text maximieren um, wenn ein Steuerelement im Strukturierten Modus dargestellt wird.

#### Automatisches Weiterbewegen

Diese Einstellung legt fest, wie sich die Braillezeile verhält, wenn der Cursor den gerade angezeigten Bereich verlässt. Wählen Sie eine der folgenden Optionen:

- **Aus ** - Diese Option schaltet das automatische Weiterbewegen aus.
- **Manuelles Weiterbewegen ausgleichen ** - JAWS bewegt die Braillezeile so weiter, wie in der Liste Benutzerdefiniertes Weiterbewegen festgelegt wurde.
- **Automatisch ** - JAWS wählt die beste Methode zur Textanzeige auf Ihrer Braillezeile.

#### Maximales Intervall beim automatischen Weiterbewegen

Legen Sie die maximale Zeitspanne (in Millisekunden) fest, die JAWS warten soll, bevor die Braillezeile weiterbewegt werden soll, wenn Sie im automatischen Modus lesen. JAWS zeigt den Text für die festgelegte Zeitspanne an und bewegt dann automatisch zum nächsten Textblock weiter. Legen Sie die Zeitspanne (in Millisekunden) fest, die JAWS warten soll, bevor die Braillezeile weiterbewegt werden soll, wenn Sie im automatischen Modus lesen. 1.000 Millisekunden sind eine Sekunde.

**Tipp:** Weitere Informationen über das Automatische Weiterbewegen finden Sie unter [JAWS und Braille Überblick](braille.chm::/braille_start.htm) und im Hilfethema Ihrer Freedom Scientific Braillezeile.

#### Braille Präsentation und Nachführung

Wählen Sie ein Element aus der Ausklappliste, um festzulegen, wie sich der Braillecursor beim Nachführen verhalten soll. Sie können diese Einstellungen als Standard oder nur für bestimmte Anwendungen festlegen. JAWS nutzt die Methoden des Off Screen Models (OSM) und des Document Object Models (DOM), um den Umfang der Braillecursorbewegung und die Steuerung beim Nachführen festzulegen. Der Vorteil des OSM besteht darin, dass der Braillecursor auf dem Bildschirm frei navigieren kann, trotzdem bietet dies für einige Anwendungen nicht genügend ausführliche Informationen. Der Vorteil des DOM besteht darin, dass dieses genaue Details bietet, aber die Braillecursornavigation auf den sichtbaren Teil des aktuellen Dokuments beschränkt.

Word und Outlook nutzen eine Mischung aus beiden Methoden, um sowohl Bildschirmdetails als auch die Navigation des Braillecursors zu maximieren.

Wählen Sie eine der folgenden OSM oder DOM Methoden, um die Braillecursornachführung für eine Anwendung oder als JAWS Standard zu steuern.

- **Immer DOM verwenden, wenn verfügbar ** - Die Verarbeitung des DOM wird genutzt, um in der Anwendung zu navigieren. Wenn DOM genutzt wird, dann ist die Navigation auf die Anwendung beschränkt. Wenn DOM nicht mit der Anwendung genutzt werden kann, dann schaltet JAWS zurück auf OSM.
- **Immer DOM verwenden, außer in Eingabefeldern ** - Die Verarbeitung des DOM wird immer zum Navigieren der Anwendung verwendet, außer, wenn der Fokus in ein Eingabefeld bewegt wird. Für das Steuerelement wird dann das OSM verwendet. Dies entspricht dem Verhalten von JAWS in Versionen vor 11.0. Dieses ist die Standardeinstellung für JAWS und andere Anwendungen, mit Ausnahme von Word und Outlook.
- **Immer DOM verwenden, außer wenn nicht beim Focus ** - Die Verarbeitung des DOM wird zum Navigieren und Darstellen genauester Informationen an der aktuellen Braillecursorposition genutzt. Wenn Sie sich von dieser Position wegbewegen, dann startet die OSM Methode und ermöglicht einfache Navigation des Braillecursors durch eine Anwendung oder über den Bildschirm. Wenn Sie eine Taste auf der Tastatur drücken, dann kehrt der Braillecursor zur aktiven Cursorposition zurück und schaltet wieder zum DOM um. Beachten Sie, dass bei aktivierter Einstellung von Aktiver Cursor folgt Braillecursor, die Nachführung innerhalb des Dokuments verbleibt, weil der aktive Cursor der Bewegung des Braillecursors folgt.
- **Immer OSM verwenden (Bildschirmpräsentation)** - Die OSM Methode überstimmt immer die DOM Ausführung, auch wenn das DOM im Dokument verwendet werden könnte.

#### Lesetasten in Braille tauschen

Aktivieren Sie dieses Kontrollfeld, um die Richtung von linker und rechter Lesetaste der Braillezeile zu tauschen. Das bedeutet, das bei Drücken einer linken Lesetaste, die Nachführung nach rechts durchgeführt wird. Ebenso wird bei Drücken einer rechten Lesetaste der Text nach links geschoben. Um die Standardnachführung wieder herzustellen, deaktivieren Sie dieses Kontrollfeld. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Aktiver Cursor folgt Braillezeile

Ist diese Einstellung aktiviert, dann werden Braillezeile und Aktive Cursor gekoppelt, so dass der aktive Cursor sich ebenfalls bewegt, wenn Sie sich auf der Braillezeile im Text weiterbewegen. Sie können den Braillecursor jedoch nicht dorthin bewegen, wo der aktive Cursor nicht hinkommt. Wenn z. B. der PC Cursor aktiv ist, können Sie mit der Braillezeile nicht bis zur Statuszeile des aktiven Fensters lesen. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Braillecursor folgt Aktivem Cursor

Aktivieren Sie dieses Kontrollfeld, so folgt der Braille Cursor dem aktiven Cursor, wenn dieser sich bewegt, ist jedoch nicht darauf beschränkt, sondern kann sich auf dem Bildschirm bewegen. So springt der Braillecursor in einem Dialogfeld immer zum aktuellen Element, wenn Sie mit TAB darauf gehen.

**Hinweis:** Im Strukturierten Modus folgt der Braillecursor immer dem aktiven Cursor, auch wenn Sie dieses Kontrollfeld nicht aktiviert haben.

### Text in 8-Punkt-Braillemodus anzeigen

Aktivieren Sie dieses Kontrollfeld, wenn Text in 8-Punkt-Braille angezeigt werden soll. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Braille Schlafmodus einschalten

Wenn aktiviert, wird der Braillezeilentreiber für diese Anwendung deaktiviert. Verwenden Sie den Braille Schlafmodus, um die Brailleausgabe in bestimmten Anwendungen zu deaktivieren. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Wortumbruch einschalten

Bei Wortumbruch trennt JAWS ein Wort nicht, das für die Darstellung auf der Braillezeile zu lang ist. Wenn Sie dann auf den nächsten Bereich weiterbewegen, können Sie das Wort im ganzen lesen. Wenn Wortumbruch ausgeschaltet ist, zeigt JAWS so viel wie möglich vom Wort an, aber ein Teil kann abgeschnitten sein. JAWS zeigt dann den Rest des Wortes im nächsten Bereich an. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Automatische Erkennung von Braillezeilen über Bluetooth

Eingeschaltet erkennt JAWS Bluetooth Braillezeilen sobald Sie sich im Empfangsbereich befinden und sendet Informationen, ohne das JAWS neu gestartet werden müsste. Die Standardeinstellung ist Aus, was bedeutet, dass Bluetooth Braillezeilen nur dann erkannt werden, wenn JAWS neu gestartet wird.

## Braillemarkierung

Nutzen Sie die Einstellungen dieser Gruppe, um festzulegen, wie Textattribute, wie fett, kursiv, unterstrichen usw. auf der Braillezeile dargestellt werden sollen.

### Braillemarkierung

Nutzen Sie die Kontrollfelder dieser Gruppe, um die Textattribute auszuwählen, die Sie innerhalb eines Textblocks auf der Braillezeile dargestellt haben möchten. Diese Attribute beinhalten hervorgehoben, fett, unterstrichen, kursiv, durchgestrichen, Farbe und skriptdefinierte Markierungen. Diese Informationen werden auf der Braillezeile unter Verwendung der **PUNKTE 7 ** und ** 8 ** angezeigt. Sind einem Textblock mehrere Attribute zugewiesen, dann zeigt der Braillecursor die Attribute nacheinander so an, wie sie in den Punktschriftmustern definiert sind, und das in der Geschwindigkeit, wie man diese unter Attributrotation eingestellt hat. Um den Attribut-Modus nutzen zu können, benötigen Sie eine Braillezeile mit Statuszellen.

Wählen Sie Braille Farbmarkierung definieren, um das Dialogfenster Braille Farbmarkierungen zu öffnen. Nutzen Sie es, um Vorder- und Hintergrundfarben für Textattribute auszuwählen. Sie müssen zuerst das Kontrollfeld Farbe aktivieren, damit Sie diesen Schalter aktivieren können. Drücken Sie **LEERTASTE **, um das Dialogfenster zu öffnen.

### Punktschriftmuster

Nutzen Sie diese Gruppe, um die Punktschriftmuster anzupassen, mit denen Textattribute wie hervorgehoben, fett, unterstrichen, kursiv, durchgestrichen, farblich und normaler Text dargestellt werden. Drücken Sie **F6 **, um direkt in das Eingabefeld zu springen und die Einstellung zu ändern. Jede Ziffer in diesem Eingabefeld bezeichnet einen Punkt auf der Braillezeile. So zeigen PUNKTE 1-2-4 beispielsweise fetten Text an. PUNKTE 1-2-5 weisen auf hervorgehobenen Text hin.

### Attrributrotationsgeschwindigkeit

Im Attributmodus werden alle Attribute, die einem Textblock zugewiesen sind, dargestellt. Wenn demselben Textblock mehrere Attribute zugewiesen sind, zeigt die Braillezeile jedes dieser Attribute nacheinander an. Nutzen Sie diese Ausklappliste, um die Geschwindigkeit der Attributsrotation in Millisekunden auszuwählen

## Cursor

Nutzen Sie die Einstellungen dieser Gruppe, um Punktschriftmuster, Cursorstatus und Blinkrate für PC Cursor, JAWS Cursor und Unsichtbaren Cursor zu konfigurieren. Wählen Sie die entsprechend Cursorgruppe, den Sie ändern möchten.

Der **PC Cursor ** ist der primär von JAWS genutzte Cursor und korrespondiert mit dem Systemfokus. Nutzen Sie ihn zum Navigieren in Menüs, zum Bearbeiten von Text in Dokumenten oder zum Navigieren in Dialogfenstern.

Der **JAWS Cursor ** ist nützlich, wenn Sie in Windows arbeiten und korrespondiert mit dem Mauszeiger. Nutzen Sie ihn, um den Mauszeiger zu bewegen, statischen Text zu lesen, um einen Überblick über die auf dem Bildschirm erscheinenden Informationen zu erhalten und um Zugriff auf andere Bereiche von Windows zu erhalten, die mit dem PC Cursor nicht erreicht werden können.

Mit dem **Unsichtbaren Cursor ** können Sie sich auf dem Bildschirm bewegen, ohne die Position des PC Cursors oder Mauszeigers zu verändern. Nutzen Sie ihn, um Text zu lesen und um auf Optionen zuzugreifen, die Sie mit dem PC Cursor nicht erreichen. Der unsichtbare Cursor kann auch dazu verwendet werden, sich einen schnellen Überblick über die momentan am Bildschirm angezeigten Informationen zu verschaffen.

### Punktmuster

Verwenden Sie dieses Eingabefeld, um das Punktmuster für den gewählten Cursor anzugeben. Jede Ziffer in diesem Eingabefeld bezeichnet einen Punkt auf der Braillezeile. Die Standardeinstellung für jeden Cursortyp ist Punkte 78. Drücken Sie **F6 **, um von der Strukturansicht direkt ins Eingabefeld zu springen.

### Cursorstatus

Diese Auswahlschalter legen fest, wie der Braillecursor angezeigt wird. Folgende Auswahlmöglichkeiten stehen zur Verfügung: Immer oben, Immer unten und Blinkend.

### Blinkrate

Dieses Eingabefeld legt fest, wie schnell der Braillecursor blinkt. Der Braillecursor blinkt in der angegebenen Zahl Millisekunden einmal an und aus. Der Standardwert ist 500. Unterschiedliche Blinkraten für jeden Cursor können helfen, zu unterscheiden, welcher Cursor gerade aktiv ist. Diese Auswahl ist nur verfügbar, wenn der Cursorstatus auf Blinkend gesetzt ist. Drücken Sie **F6 **, um von der Strukturansicht direkt in das Eingabefeld zu springen.

## Blitzmeldungen

Braille Blitzmeldungen sind kurze Beschreibungen, die auf Ihrer Braillezeile ein paar Sekunden lang erscheinen. Die Meldungen können Fehlermeldungen, Statusinformationen, Hilfe-Sprechblasen und andere Informationen beinhalten. Sie können diese Meldungen jederzeit unterdrücken, indem Sie eine Cursorroutingtaste drücken. Wenn Sie mehr Zeit benötigen, die Meldung zu lesen, drücken Sie auf Ihrer Braillezeile die Navigationstaste, um die Meldung länger anzuzeigen.

### Anzeigedauer für Blitzmeldungen

Wählen Sie eine Zeitspanne (in Millisekunden) aus, in der die Blitzmeldungen auf der Braillezeile stehen bleiben sollen. Sie haben die Wahl zwischen 0 und 30.000 Millisekunden. 1.000 Millisekunden sind eine Sekunde.

**Tipp:** Wählen Sie "Nur manuell löschen", wenn JAWS die Meldung so lange auf der Zeile stehen lassen soll, bis Sie eine Cursorroutingtaste gedrückt haben.

### Blitzmeldungen einschalten

Aktivieren Sie dieses Kontrollfeld, wenn JAWS Informationen mithilfe von Braille Blitzmeldungen anzeigen soll. Deaktivieren Sie dieses Kontrollfeld, wenn keine Blitzmeldungen angezeigt werden sollen.

### Präfix für eine Meldung anzeigen

Wenn Sie dieses Kontrollfeld aktivieren und Ihre Braillezeile über keine Statuszellen verfügt, dann zeigt JAWS ein spezielles Präfix vor Blitzmeldungen an, damit man diese von normalem Bildschirmtext unterscheiden kann. Dieses Präfix ist eine Abkürzung mit drei Buchstaben, das die Art der Meldung anzeigt. Sofern Ihre Braillezeile über Statuszellen verfügt, verwendet JAWS diese, um anzuzeigen, dass es sich bei dem Text um eine Blitzmeldung handelt, egal ob dieses Kontrollfeld aktiviert ist oder nicht. Deaktivieren Sie dieses Kontrollfeld, wenn Sie diese Funktion nicht nutzen möchten.

### Ausführlichkeitsstufen

Markieren Sie einen der folgenden Auswahlschalter im Gruppenfeld Ausführlichkeitsstufen, um festzulegen, was JAWS als Blitzmeldung anzeigen soll:

- **Einsteiger, Höchste ** - Auf dieser Stufe zeigt JAWS über die Blitzmeldungen die Startmeldungen von Anwendungen, Fehlermeldungen, Hilfe-Sprechblasen, JAWS Meldungen, Smart-Hilfemeldungen, Statusinformationen und vom Benutzer angeforderte Informationen standardmäßig an.
- **Fortgeschritten ** - Auf dieser Stufe zeigt JAWS über die Blitzmeldungen Fehlermeldungen, Hilfe-Sprechblasen, JAWS Meldungen, Smart-Hilfemeldungen, Statusinformationen und vom Benutzer angeforderte Informationen standardmäßig an.
- **Experten, Niedrigste ** - Auf dieser Stufe zeigt JAWS über die Blitzmeldungen Fehlermeldungen, Hilfe-Sprechblasen, Statusinformationen und vom Benutzer angeforderte Informationen standardmäßig an.

### Ausführlichkeitsstufen anpassen

Über die Schalter Einsteiger Einstellungen, Fortgeschrittenen Einstellungen und Experten Einstellungen werden Dialogfelder geöffnet, in denen Sie jede Ausführlichkeitsstufe anpassen können. Alle Dialogfelder enthalten folgende Optionen:

#### Braille-Elemente

In der Liste der Braille-Elemente wählen Sie aus, was als Blitzmeldung angezeigt werden soll. Die Auswahl basiert auf [Sprachausgabe-Typen](../SpeechVerbosity/speech_output_types.htm) und beinhaltet Startmeldungen von Anwendungen, Fehlermeldungen, Hilfe-Sprechblasen, JAWS Meldungen, Smart-Hilfemeldungen, Statusinformationen, Tool Tipps, Tutormeldungen und vom Benutzer angeforderte Informationen. In dieser Liste finden Sie auch die Abkürzungen, die JAWS jeder Blitzmeldung als Präfix voranstellt.

#### Länge der JAWS Meldungen

Im Gruppenfeld Länge der JAWS Meldungen haben Sie die Wahl zwischen kurzen oder langen Meldungen. Wenn Sie zum Beispiel Text mit **STRG+C ** in die Zwischenablage kopiert haben, zeigt JAWS "Kopiert" an, wenn Sie kurze Meldungen verwenden. Bei langen Meldungen zeigt JAWS "Ausgewählter Text in die Zwischenablage kopiert" an.

#### Brailletext modifizieren

Aktivieren Sie den Schalter Brailletext modifizieren, wenn Sie das Präfix einer Meldung verändern möchten. So ist zum Beispiel das Präfix für eine Hilfe-Sprechblase "bln" (vom englischen Balloon). Markieren Sie die Abkürzung, die Sie verändern möchten, in der Liste und aktivieren Sie dann Brailletext modifizieren. Geben Sie dann die neue Abkürzung ein und drücken auf OK.

#### Standardeinstellungen wiederherstellen

Der Schalter Standardeinstellungen wiederherstellen löscht alle Angaben, die auf die aktuelle Ausführlichkeitsstufe in der geladenen JCF-Datei zutreffen. ## Die Gruppe Formularmodus

Der Formularmodus wird benötigt, wenn Sie mit HTML Inhalten in einem Browser arbeiten und Sie Daten eingeben müssen. Seit JAWS zum Navigieren einen "Virtuellen Modus" verwendet, können Sie normalerweise einen Buchstaben zum Navigieren zwischen Überschriften, Absätzen, Tabellen, etc drücken. Aber wenn Sie Text eingeben müssen oder die **PFEILTASTEN ** zum Auswählen von Elementen in einer Ausklappliste verwenden möchten, dann müssen Sie auf einen anderen Modus wechseln, den Formularmodus. Nutzen Sie die Einstellungen dieser Gruppe, um den Formularmodus zu konfigurieren Formulareinstellungen finden Sie in der Gruppe Formularmodus in der Einstellungsverwaltung.

## Formularmodus wählen (Automatisch, Halbautomatisch oder Manuell)

Nutzen Sie die Auswahlschalter in dieser Gruppe, um festzulegen, wie Sie den Formularmodus aktivieren und diesen auch wieder verlassen.

Ist dieser auf Automatisch gesetzt, dann aktiviert JAWS den Formularmodus automatisch, wenn Sie **TAB **, ** UMSCHALT+TAB ** oder die ** PFEILTASTEN ** drücken, um zu einem Element zu springen, welches den Formularmodus benötigt, damit Sie mit dem Element arbeiten können, wie etwa bei Eingabefeldern oder Ausklapplisten. Um anzuzeigen, dass Sie den Formularmodus aktiviert haben, hören Sie einen Pop Sound, oder nutzen Sie dazu die Statuszellen Ihrer Braillezeile. Wenn Sie das Arbeiten mit einem Element abgeschlossen haben, dann können Sie ** ESC ** drücken, um den Formularmodus zu verlassen, oder drücken Sie ** PFEIL RAUF ** oder ** RUNTER **, um vom Element wegzuspringen und den Formularmodus zu beenden. Diese Funktionalität kann für neue und sehende JAWS Anwender sehr nützlich sein und ist daher die Standardeinstellung.

Die Auswahl von Manuell verhindert, dass JAWS den Formularmodus immer aktiviert, sobald Sie mit Befehlen inklusive der **TAB ** Taste zwischen Elementen navigieren. Das bedeutet, dass Sie beim Springen auf ein Eingabefeld oder eine Ausklappliste zuerst ** EINGABE ** drücken müssen, um Text einzugeben oder einen Wert auszuwählen. Um den Formularmodus manuell zu beenden, drücken Sie ** NUM PLUS ** oder ** ESC **.

Halbautomatisch liegt zwischen Automatisch und Manuell. Der Formularmodus wird aktiviert, wenn Sie beim Navigieren mit **TAB ** oder ** UMSCHALT+TAB ** auf einem Element landen. Der Formularmodus wird nicht aktiviert, wenn Sie mit den ** PFEILTASTEN ** oder den Schnellnavigationstasten navigieren. Sie müssen den Formularmodus manuell beenden, wenn Halbautomatisch aktiviert ist. ## Ansage von speziellen Symbolen bei

Nutzen Sie diese Liste, um festzulegen, wann JAWS den numerischen Wert eines speziellen Symbols, wie Copyright, Registriert und Warenzeichen ansagen soll, die standardmäßig ignoriert werden.

- **Nie **: JAWS liest bestimmte Zeichen mit der aktuell ausgewählten Sprachausgabe und sagt nicht deren numerischen Wert an.
- **Tastaturecho, Zeichenansage, Buchstabieren **: JAWS sagt den numerischen Wert bestimmter Zeichen bei der Eingabe, beim zeichenweisen Lesen (** PFEIL LINKS ** oder ** PFEIL RECHTS **), oder beim Buchstabieren eines Wortes (** JAWS TASTE+NUM 5 ** zweimal hintereinander) an.

Wenn Sie dieses Kontrollfeld aktivieren, spricht JAWS den Hexadezimalwert des aktuellen Zeichens, und nicht den ASCII Wert, sobald Sie **NUM 5 ** dreimal schnell hintereinander drücken. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Zeichenwert als Sequenz von Multibyte-Werten sprechen

Diese Option legt fest, wie JAWS den Wert von Unicode-Zeichen spricht. Wenn Sie dieses Kontrollfeld deaktivieren, spricht JAWS den Wert des Zeichens so wie er ist (zum Beispiel "u+627 hex"). Wenn Sie dieses Kontrollfeld aktivieren, wandelt JAWS das Zeichen in eine Sequenz um (abhängig von der aktuell aktiven Codepage) und spricht den Wert jedes einzelnen Bestandteils des Zeichens. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

Es bietet eine vorhersehbare Navigation durch die Multifunktionsleiste einer Anwendung, eine Art Menü, das in Windows-Anwendungen wie Microsoft Office üblich ist. Das Virtuelle Menüband organisiert alle Menübänder in eine Menüstruktur, wodurch eine Navigation mit **PFEILTASTEN ** sehr einfach gemacht wird. Ist die Einstellung eingeschaltet, dann kann man im Menüband wie in traditionellen Menüs oder Untermenüs navigieren, so wie es den meisten JAWS Anwendern vertraut ist. Dies gilt für alle Anwendungen, die das Menübandformat verwenden.

Um zwischen den Registern des oberen Menübands zu wechseln, drücken Sie **PFEIL LINKS ** oder ** PFEIL RECHTS **.

Um zum unteren Menüband des aktuellen Registers zu springen, drücken Sie **PFEIL RUNTER **.

Um das obere Menüband zu verlassen und zum Dokument zurückzukehren, drücken Sie **ESC ** oder ** ALT **.

Steht für eine Anwendung eine Tastenkombination von **ALT-Taste ** kombiniert mit einem Buchstaben zum Aufruf eines Befehls zur Verfügung, dann nutzt JAWS die Kurztaste und das Virtuelle Menüband erscheint nicht.

Die Einstellung Virtuelles Menüband verwenden steht nur zur Verfügung, wenn Sie die Standardeinstellung in der Ausklappliste für Anwendungen gewählt haben. Sie steht nicht zur Verfügung, wenn Sie andere anwendungsspezifische Einstellungen verwenden. Das Kontrollfeld Virtuelles Menüband verwenden ist standardmäßig nicht aktiviert. Beachten Sie, dass Sie das Virtuelle Menüband auch über den Startassistenten ein- oder ausschalten können.

## Virtuellen PC Cursor verwenden

Wenn dieses Kontrollfeld aktiviert ist, wird der virtuelle Cursor zum Lesen von HTML Dokumenten verwendet. Bei Nutzung des Virtuellen PC Cursors werden Webseiten und andere HTML Dokumente gelesen, als wären Sie in einzelne Spalten formatiert, mit jedem Link oder Formularfeld in einer eigenen Zeile. Dieser Modus ermöglicht das Navigieren und Markieren von Text auf einer Webseite als wären Sie in einem Standard-Textdokument, während Sie weiterhin die Möglichkeit haben, Formulare auszufüllen und Links zu aktivieren. Dieses Kontrollfeld ist standardmäßig aktiviert.

## Virtueller Cursor An/Aus abhängig von Browser Seiten

Ist dieses Kontrollfeld aktiviert, dann merkt sich JAWS den aktuellen Status des virtuellen Cursors, wenn dieser mit **EINFÜGEN+Y ** für eine Seite in Google Chrome umgeschaltet wird. Wenn Sie beispielsweise den virtuellen Cursor in einer Webanwendung, wie etwa GMail, welche dann eine bessere Unterstützung bietet, ausschalten, und Sie dann ** STRG+N ** oder ** STRG+T ** zum Öffnen einer Webseite in einem neuen Fenster drücken, dann wird der virtuelle Cursor für das neue Fenster wieder eingeschaltet, bleibt aber weiterhin in GMail deaktiviert. Hierdurch wird ermöglicht, dass Sie mit dem Fokus wieder zurück zu GMail springen können und nicht mehr dran denken müssen, den virtuellen Cursor dort wieder zu deaktivieren. Der virtuelle Cursor kehrt wieder zu seiner Standardeinstellung zurück, wenn Sie die Browserseite schließen oder JAWS neu starten.

Das Deaktivieren dieses Kontrollfelds bewirkt, dass der virtuelle Cursor wieder zu seiner Standardeinstellung zurückkehrt sobald sich der Fokus ändert, so wie er es in den vorherigen Versionen getan hat. Wenn Sie ihn beispielsweise in GMail ausschalten und dann **ALT+TAB ** drücken, um in ein anderes Fenster zu wechseln, dann wird der virtuelle Cursor wieder aktiviert, sobald der Fokus auf die GMail Seite zurückkehrt.

Dieses Kontrollfeld ist standardmäßig aktiviert. Aus technischen Gründen funktioniert diese Funktion zur Zeit nur in Chrome.

## Verzögerung bei Textausgabe

Dieses Eingabefeld gibt die Anzahl von Millisekunden an, die JAWS wartet, bevor es neu auf den Bildschirm geschriebenen Text spricht. Der Standardwert ist 0.

## Pixel pro Leerschritt

JAWS muss die Anzahl von Leerzeichen basierend auf leeren Bildschirmbereichen berechnen. JAWS basiert solche Berechnungen auf der Anzahl Bildpunkte, die hier angegeben werden. Der Standardwert ist 8.

## Pixel pro Tab

JAWS kann die Anzahl der Tabs ermitteln, die in einem leeren Abschnitt einer Zeile enthalten sind. JAWS basiert solche Berechnungen auf der Anzahl der Pixel, die hier angegeben werden. Der Standardwert ist 10.

## Linker Rand in Pixel

JAWS ignoriert die angegebene Anzahl der Pixel vom linken Rand des Fensters. Dies kann JAWS davor bewahren, den linken Rand falsch zuzuordnen, oder Leerzeichen zu lesen, wo in Wirklichkeit gar keine vorhanden sind.

## Abstand beim Unterstreichen

In diesem Eingabefeld legen Sie fest, wie weit ein Unterstrich vom Text entfernt sein muss, damit JAWS ihn als Unterstrich betrachtet. Der Standardwert ist 0 Pixel.

## Accessibility Treiber für Bildschirmauswertung verwenden

Der Zugänglichkeitstreiber ist ein Treiber auf Systemebene, der durch unsere Software installiert wird und der seit vielen Jahren die Bildschirminformationen an JAWS übergibt. Diese Information ist als Off Screen Model (OSM) bekannt und wird hauptsächlich vom Lesecursor, wie etwa JAWS Cursor und Unsichtbarer Cursor genutzt, aber auch von JAWS Skripts. Da Windows zunehmend unterschiedliche Methoden zur Anzeige von Informationen verwendet, funktioniert dieser Zugänglichkeitstreiber in vielen Fällen nicht mehr.

Ein Deaktivieren dieses Kontrollfelds konfiguriert JAWS so, dass die neue Technologie der Zugänglichkeit auf Anwendungsebene verwendet wird, die wir entwickelt haben, um die fehlenden Informationen des Zugänglichkeitstreibers aufzufangen.

Sie können dem Skript ToggleScreenCaptureMethod auch eine Kurztaste über den Tastaturmanager zuweisen, um diese Einstellung schnell umzuschalten.

Dieses Kontrollfeld ist in Windows 10 standardmäßig aktiviert, aber in Windows 11 standardmäßig deaktiviert.

## Fehlermitteilung

Nutzen Sie die Auswahlschalter dieser Gruppe, um festzulegen, ob JAWS Fehlerberichtdateien an Freedom Scientific senden soll oder nicht, sobald JAWS nach einem unerwartetem Fehler wieder gestartet wird. Sie können wählen zwischen:

- **Jedesmal nachfragen, ob ich einen Fehlerbericht an Freedom Scientific senden möchte **: JAWS zeigt jedes Mal ein Dialogfenster an, wenn ein Fehler festgestellt wurde und fragt, ob Sie die will Informationen an Freedom Scientific senden möchten. Wählen Sie den Schalter Senden, um den Fehlerbericht direkt an Freedom Scientific zu übermitteln, oder Wählen Sie Nicht senden, wenn Sie nicht möchten, dass ein Fehlerbericht an Freedom Scientific gesendet wird. Dies ist die Standardeinstellung.
- **Alle Fehlerberichte an Freedom Scientific senden **: Wenn Sie diese Einstellung wählen, dann werden zukünftige Fehlerberichte automatisch ohne Nachfrage an Freedom Scientific gesendet.
- **Niemals Fehlerberichte an Freedom Scientific senden **: Wenn Sie diese Einstellung wählen, dann werden zukünftige Fehlerberichte nicht an Freedom Scientific gesendet und Sie werden auch nicht zum Senden eines Berichts aufgefordert.

**Hinweis:** Die an Freedom Scientific übermittelten Informationen sind beschränkt auf eine Datei des Speicherabbilds, und Informationen, die es Freedom Scientific ermöglichen, den Fehlergrund zu untersuchen, wie beispielsweise Ihre Produktversionsnummer, Zeit und Datum des Fehlerauftretens, die Anzahl der von Ihrem System festgestellten Fehler und dem Fehlercode, der zum aufgetretenen Problem gehört. Die einzige übermittelte Information, die sich auf Ihre spezielle Computerausstattung bezieht, ist die MAC Adresse Ihres Rechners, um Trends auf einzelnen Systemen feststellen zu können. Allerdings gibt es für uns keine Möglichkeit, von MAC Adressen Rückschlüsse auf bestimmte Personen zu ziehen, da keinerlei persönliche Identifizierungsinformationen übermittelt werden.

## Die Gruppe Mausecho

Sie können JAWS so konfigurieren, dass Text gelesen wird, wenn Sie mit dem Mauszeiger darauf zeigen. Nutzen Sie die Einstellungen dieser Gruppe, um Einstellungen für das Mausecho zu konfigurieren.

## Mausecho einschalten

Aktiviert spricht JAWS Text, wenn Sie mit dem Mauszeiger drauf zeigen. Das Mausecho ist standardmäßig deaktiviert.

## Mausecho unterbricht das Sprechen

Aktivieren Sie dieses Kontrollfeld, damit JAWS die Sprachausgabe unterbricht, wenn Sie den Mauszeiger über Text bewegen. Wenn Sie dieses Kontrollfeld deaktivieren, dann können Sie den Mauszeiger über den gesamten Bildschirm bewegen, ohne dass die Sprachausgabe unterbrochen wird. Dieses Kontrollfeld ist standardmäßig aktiviert.

## Unverzögertes Mausecho

Aktivieren Sie dieses Kontrollfeld, damit Text gelesen wird, während sich der Mauszeiger bewegt und Sie nicht erst warten müssen, um den Text unter dem Mauszeiger zu hören. Das Deaktivieren dieses Kontrollfelds bewirkt, dass JAWS für die in der Ausklappliste Mausechoverzögerung festgelegte Dauer wartet, bevor der Text unter dem Mauszeiger gelesen wird. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Mausecho Verzögerung

Diese Einstellung gibt Ihnen die Möglichkeit, die Dauer anzugeben, die JAWS warten soll, bevor Text an der Mauszeigerposition gelesen wird. Die Verzögerung wird in Sekunden gemessen. Wenn Sie den Mauszeiger weiterbewegen, bevor die Verzögerung abgelaufen ist, dann wird JAWS den Text nicht lesen. Der Standardwert der Mausecho-Verzögerung ist 0,4 Sekunden. Diese Einstellung steht nicht zur Verfügung, wenn das Kontrollfeld Unverzögertes Mausecho aktiviert ist, was bedeutet, dass Text gelesen wird, während die Maus in Bewegung ist, so dass Sie nicht warten müssen, um den Text unter dem Mauszeiger zu hören.

## Mausecho Einheit

Wählen Sie die Texteinheit, die Sie hören möchten, wenn der Mauszeiger auf dem Text eines Steuerelements stehen bleibt. Die verfügbaren Einstellungen sind Zeichen, Wort, Zeile oder Absatz. Die Standardeinheit für das Mausecho ist Zeile. Das bedeutet, dass wenn Sie den Mauszeiger in den Dokumentbereich einer Anwendung bewegen, wie etwa in Microsoft Word, dass JAWS die Textzeile unter dem Mauszeiger vorliest. Wenn Sie die Maus im Eingabebereich weiterbewegen, dann liest JAWS andere Zeilen des Dokuments.

Wenn das Steuerelement nicht über die angeforderte Texteinheit verfügt, dann liest JAWS den gesamten Text, den das Steuerelement bietet. Wenn Sie beispielsweise die Maus in den Eingabebereich der Windows 10 App Sticky Notes bewegen, dann liest JAWS den gesamten Inhalt des Eingabefeldes, anstatt nur die angegebene Texteinheit.

## Elementbeschreibung ansagen

Diese Informationen werden nach dem eigentlichen Text unter dem Mauszeiger angesagt. Beachten Sie, dass der beschreibende Text nicht immer zur Verfügung steht. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## JAWS Konfigurationsdateien

Die JAWS Konfigurationsdateien (JCF) speichern Einstellungen für sowohl JAWS als auch für andere Softwareanwendungen. Der vollständige Name der Konfigurationsdatei wird durch den Anwendungsnamen festgelegt, für die diese Datei erstellt wird. So ist z. B. der Name der ausführbaren Datei von Word "Microsoft Word.exe", so dass der Name der Konfigurationsdatei für Word "Microsoft Word.jcf" lautet.

Eine wichtige JAWS Funktion besteht darin, Einstellungen entweder global oder anwendungsbezogen zu speichern. Anwendungsbezogene Einstellungen werden gespeichert, indem der Name der Anwendung und die Erweiterung.jcf verwendet werden, wie zum Beispiel "Microsoft Word.jcf". Die globalen Einstellungen werden in der Konfigurationsdatei Default.jcf gespeichert. Anwendungsabhängige Einstellungen können globale Einstellungen überschreiben, wenn Sie also eine Änderung in Default.jcf vornehmen, diese Änderung aber nicht in allen Anwendung Verwendung findet, müssen Sie die Änderungen in den entsprechenden JCF-Dateien ebenfalls vornehmen.

### Gemeinsam genutzte Dateien

JAWS erlaubt den Zugriff auf Benutzer- und gemeinsam genutzte Konfigurationsdateien. JAWS weist die Einstellungen der Benutzer-Konfigurationsdateien nur dann zu, wenn sich der entsprechende Benutzer an dem Computer einloggt. Die Einstellungen in gemeinsam genutzten Dateien werden allen Benutzern des Computers oder Netzwerks zugewiesen. Dadurch werden Systemadministratoren in die Lage versetzt, JAWS für alle Benutzer eines Netzwerks oder Computers entsprechend zu konfigurieren. Wenn ein Benutzer eine gemeinsam genutzte Datei verändert, dann werden die Einstellungen überschrieben, während der Benutzer am Computer eingeloggt ist.

Wenn Sie im Dateimenü den Befehl Öffnen ausführen, können Sie "Gemeinsame Konfigurationen" im Listenfeld Suchen in auswählen, um eine gemeinsam genutzte Datei zu suchen, die Sie überprüfen oder modifizieren können. JAWS speichert gemeinsame Konfigurationsdateien in ProgramData\Freedom Scientific\JAWS\*Versionsnummer *\Settings\Deu.

**Tipp:** Sie können sehr schnell Benutzer- und gemeinsam genutzte Dateien überprüfen, indem Sie die Verknüpfungen im Eintrag "JAWS erkunden" in der JAWS Programmgruppe verwenden.

# Gruppenfeld Alles lesen

## Alles lesen liest

Wenn Sie Alles lesen einsetzen (**JAWS TASTE+PFEIL RUNTER **), dann fügt JAWS eine kleine Pause zwischen Textpassagen wie zum Beispiel Sätzen oder Absätzen ein. Verwenden Sie die Auswahlschalter in dieser Gruppe, um festzulegen, wo und wie JAWS bei Alles lesen eine Pause einlegen soll.

- **Zeilenweise mit Pausen **: Diese Option liest zeilenweise und legt am Ende einer jeder Zeile eine kurze Pause ein.
- **Zeilenweise ohne Pausen **: Diese Option erlaubt ein flüssiges Lesen, ohne Pausen am Zeilenende.
- **Satzweise **: Aktivieren Sie diesen Auswahlschalter für alle Anwendungen, in denen das Vorlesen abgehackt klingt. Diese Einstellung setzt die Pause zwischen Elementen ans Ende eines jeden Satzes und macht das Vorlesen flüssiger.
- **Absatz **: Diese Option ist besonders in großen Dokumenten nützlich, weil Sie so mit der rechten ** UMSCHALTTASTE ** schnell durch die Absätze springen können, ohne das Vorlesen unterbrechen zu müssen.

## Schema auswählen

Dies ermöglicht es, ein unterschiedliches Schema zu laden, das während des Alles Lesens verwendet wird. Dies bezieht sich nur auf HTML und Word Dokumente. Das Schema Alles Lesen mit Sounds spricht den Text von Links und Überschriften, aber spielt nur einen unaufdringlichen Sound, anstatt der Ansage von Link und Überschrift. Das Schema Nur Alles Lesen gibt keinen Hinweis auf Links oder Überschriften, es sagt lediglich den Text von Links oder Überschriften an. Beachten Sie, dass Sie auch den Schalter Hinzufügen/Entfernen nutzen können, um andere Schemata, die für Alles Lesen genutzt werden können, zu ändern oder zu löschen.

## Hinzufügen/Entfernen

Wählen Sie Hinzufügen/Entfernen, um das Dialogfenster Schema für Alles Lesen auswählen zu öffnen. Es beinhaltet eine Liste der vorinstallierten Sprach- und Soundschemata, die Sie der Gruppe Alles Lesen Schema hinzufügen oder von dieser entfernen können. Um ein Schema zur Liste hinzuzufügen, aktivieren Sie das dazugehörige Kontrollfeld und drücken Sie OK. ## Leerzeilen während Alles Lesen ansagen

## Diverses

Diese Seite enthält folgende unterschiedliche Elemente, denen Sie ein Verhalten zuordnen können:

- **Alles Groß ** - Bezieht sich auf Text, der nur in Großbuchstaben erscheint. Das diesem Element zugewiesene Verhalten hängt auch von dem korrespondierenden Element im Dialogfeld Textverarbeitung ab.
- **Rahmen ** - Bezieht sich auf Text mit einer Umrandung.
- **Großschreibung ** - Bezieht sich auf Großbuchstaben. Das diesem Element zugewiesene Verhalten hängt auch von dem korrespondierenden Element im Dialogfeld Textverarbeitung ab.
- **Neue Zeile ** - Dieses Verhalten tritt dann auf, wenn Sie in eine neue Textzeile springen.
- **Neuer Absatz ** - Dieses Verhalten tritt dann auf, wenn Sie an den Anfang eines neues Absatzes springen.
- **Zitat ** - Bezieht sich auf zitierten Text.
- **Wiederholung ** - Dieses Verhalten tritt dann auf, wenn JAWS sich wiederholende Zeichen erkennt.
- **Schattierung ** - Bezieht sich auf Text mit einer Schattierung.
- **Buchstabieren ** - Dieses Verhalten wird angewendet, wenn JAWS ein Wort buchstabiert.

## Verfügbare Verhaltensweisen

Jede der folgenden Verhaltensweisen kann zur Verfügung stehen, wenn Elemente im Dialog Schema bearbeiten modifiziert werden. Nicht alle Optionen stehen für jedes Element zur Verfügung. Der eigentliche Einfluss einer Verhaltensweise kann für einige Elemente unterschiedlich sein.

### Element sprechen

Wenn dieses Verhalten ausgewählt wird, spricht JAWS dieses Element ganz normal. Sie können jedoch ein Stimmenalias und/oder ein anderes Wort oder eine andere Wendung, die gesprochen werden soll, auswählen.

### Sounds abspielen

Wenn dieses Verhalten ausgewählt wird, spielt JAWS die festgelegte Sounddatei ab, anstatt das ausgewählte Element zu sprechen. Der Sound muss im Format.wav sein und sich im Unterverzeichnis Sounds Ihres JAWS Verzeichnisses Settings befinden. Wenn Sie den Schalter Wave-Datei auswählen aktivieren, erscheint ein Dialog, in dem Sie sich einen Sound aussuchen können, der abgespielt werden soll. Wenn Sie in einem anderen Verzeichnis einen Sound auswählen, kopiert JAWS diese Sounddatei automatisch in das Unterverzeichnis Sounds. Mehrere Sounddateien sind in JAWS enthalten.

### Stimme verändern

Wenn dieses Verhalten ausgewählt wird, spricht JAWS das angezeigte Element nicht als zusätzliche Information, die dem sichtbaren Text hinzugefügt wurde, wie zum Beispiel der Typ des Steuerelements oder ein Textattribut. Statt dessen spricht JAWS den eigentlichen Text mit dem von Ihnen festgelegten Stimmenalias.

### Sprache auswählen

Wenn dieses Verhalten ausgewählt wird, spricht JAWS das angezeigte Element nicht als zusätzliche Information, die dem sichtbaren Text hinzugefügt wurde, wie zum Beispiel ein Textattribut. Statt dessen spricht JAWS den eigentlichen Text in der von Ihnen festgelegten Sprachausgabesprache.

### Ignorieren

Wenn dieses Verhalten ausgewählt wird, ignoriert JAWS die Existenz des festgelegten Elements, aber nicht den zugewiesenen Text. Dies ist häufig in Kombination mit anderen Regeln hilfreich, wenn Sie sich einiger Veränderungen bewusst sein möchten, aber keine anderen.

## Sprach- und Sound-Schemata

# Die Gruppe Sprach- und Sound-Schemata

Nutzen Sie die Gruppe Sprach- und Soundschemata, um anzupassen, wie JAWS Sie über verschiedene Arten von Informationen unterrichten soll. Sie können zum Beispiel ganz einfach festlegen, dass JAWS eine ansteigende Tonfolge abspielen soll, wenn man auf ein aktiviertes Kontrollfeld springt, eine absteigende Tonfolge ertönen soll, wenn ein Kontrollfeld erreicht wird, das nicht aktiviert ist, und dass JAWS immer dann "Hier tippen" sagen soll, wenn ein Eingabefeld betreten wird. Arbeiten Sie mit Dokumenten, die Text in einer anderen Sprache enthalten, und der zudem immer fett gesetzt ist, dann kann JAWS auf eine bestimmte Sprache umschalten, wenn es auf fett gesetzten Text trifft. Es gibt viele Möglichkeiten, immer abhängig von Ihren Anforderungen.

Sie können Ihre Sprach- und Sound-Einstellungen als Schema speichern. Schemata sind Auflistungen von Regeln des Sprachmanagers, gespeichert als.smf Dateien, im Ordner C:\Users\*B **enutzerName *\AppData\Roaming\Freedom Scientific\JAWS\* Versionsnummer *\Settings\DEU. Sie können mehrere Schemata erstellen und speichern, aber nur ein Schema kann zurzeit aktiv sein.

Um Sprach- und Soundeinstellungen zu ändern, öffnen Sie die Gruppe Sprach- und Soundschemata in der Einstellungsverwaltung.

## Schemata auswählen, bearbeiten und erstellen

JAWS verfügt über mehrere Standard-Schemata. Das Klassik-Schema bewirkt, dass JAWS sich wie gewohnt verhält, und ist somit ideal für den Anfang. Die anderen Schemata verwenden das Klassik-Schema als Grundlage und fügen zusätzliche Informationen hinzu, die zum Beispiel für das Korrekturlesen von Dokumenten nützlich sein können Im Sprach- und Soundmanager wählen Sie Schemata aus, bearbeiten, speichern oder erstellen sie. Zum Beispiel:

- Um ein anderes Schema zu aktivieren, wählen Sie es in der Liste aus. Drücken Sie **JAWS TASTE+T ** zum Lesen der Titelzeile, und JAWS meldet das aktive Schema. Um [das aktive Schema zu bearbeiten](edit_scheme.htm), gehen Sie mit ** TAB ** auf den Schalter Aktuelles Schema bearbeiten und aktivieren diesen. Nachdem Sie ein Schema bearbeitet haben, speichern Sie die Änderungen, indem Sie den Schalter Aktuelles Schema speichern aktivieren.
- Um eine Kopie des aktuellen Schemas unter einem anderen Namen zu speichern, aktivieren Sie den Schalter Aktuelles Schema speichern unter. Dadurch wird eine Kopie des aktiven Schemas mit einem Namen Ihrer Wahl erzeugt, so dass Sie Veränderungen am neuen Schema vornehmen können, ohne das alte zu verändern.
- Um ein neues Schema zu erstellen, aktivieren Sie den Schalter Neues Schema erstellen. Es wird ein Schema basierend auf den Einstellungen des Klassik-Schemas erstellt. Um das neue Schema zu verändern, markieren Sie es in der Liste und aktivieren den Schalter Aktuelles Schema bearbeiten. Lesen Sie auch das Hilfethema [Der Dialog Schema bearbeiten](edit_scheme.htm), um weitere Informationen über das Erstellen und Verändern von Schemata zu erhalten.
- Um ein Schema zu löschen, markieren Sie es in der Liste und aktivieren dann den Schalter Schema löschen. Das Klassik-Schema kann nicht gelöscht werden.

**Tipp:** Verwenden Sie den Sprach- und Sound- Lernmodus, um zu erfahren, wie das aktive Schema Klänge verwendet, um Informationen zu liefern. Lesen Sie weiter unten das Thema [Trainingsmodus](#train_mode) für weitere Informationen.

## Schemata bei laufendem Betrieb verändern

Neben der Möglichkeit, Schemata dauerhaft in der Einstellungsverwaltung zu ändern, bietet JAWS die Möglichkeit an, Schemata schnell und temporär zu ändern. Dazu drücken Sie **ALT+JAWS TASTE+S **, markieren das entsprechende Schema und drücken ** EINGABE **.

## Sprach- und Sound-Lernmodus

Der Sprach- und Sound- Lernmodus hilft Ihnen zu verstehen, wie ein Schema Klänge verwendet, um Sie mit Informationen zu versorgen. Bei eingeschaltetem Sprach- und Sound- Lernmodus liefert JAWS detaillierte Informationen über Elemente, Status und Attribute und auch über die zugewiesenen Klänge in dem aktiven Schema. Sie hören den Klang, danach spricht JAWS die gleiche Information, die Sie beim Einsatz des Schemas "Klassisch" hören.

**Tipp:** Der Sprach- und Sound- Lernmodus liefert auch Informationen über Elemente, Status und Attribute, die vom Schema nicht berücksichtigt werden.

Falls der vom Ersteller des Schemas zugewiesene Klang nicht eindeutig zuzuordnen ist, kann man mit Hilfe des Sprach- und Sound- Lernmodus den Zweck eines jeden Klanges ermitteln. Wenn ein Schema zum Beispiel zwei unterschiedliche Klänge zum Anzeigen eines Kontrollfeld-Status' verwendet, können Sie möglicherweise nicht gleich erkennen, welcher Klang die Aktivierung des Kontrollfeldes meldet. Bei aktivem Sprach- und Sound- Lernmodus spielt JAWS den Klang ab und sagt dann "aktiviert" oder "nicht aktiviert".

Ein- und Ausschalten des Sprach- und Sound- Lernmodus:

1. Drücken Sie **ALT+JAWS TASTE+S **.
2. Um den Lernmodus einzuschalten, wählen Sie "Sprach- und Sound- Lernmodus an". Um den Lernmodus auszuschalten, wählen Sie "Sprach- und Sound- Lernmodus aus".
3. Drücken Sie **EINGABE **.

- Intelligente Hilfemeldungen
- Hilfemeldungen

## Die Gruppe Sprachausführlichkeit

Nutzen Sie die Einstellungen dieser Gruppe, um die Ausführlichkeitsstufen anzupassen. Die Ausführlichkeitsstufe legt fest, wieviel Informationen von JAWS gesprochen werden. Dies wird auch in der Titelzeile angezeigt, wie zum Beispiel Einsteiger Einstellungen.

Um die Sprachausführlichkeitseinstellungen zu öffnen, gehen Sie wie folgt vor:

Ist die Datei Default.jcf geladen, werden Werkseinstellungen wiederhergestellt. Obwohl das Dialogfeld diese Änderung widerspiegelt, werden die wiederhergestellten Standardeinstellungen erst wirksam, wenn Sie die JCF-Datei speichern.

**Hinweis:** Wenn Sie einen der Dialoge Einstellungen schließen, hängt es von der Methode ab, mit der Sie den Dialog Einstellungen geöffnet haben, wo der Fokus im Dialog Ausführlichkeitsoptionen landet. Wenn Sie einen der Dialoge Einstellungen im Dialog Ausführlichkeitseinstellungen mit einer Tastenkombination geöffnet haben, landen Sie wieder auf dem Element, welches vorher den Fokus hatte. Wenn Sie mit ** TAB ** zu dem Schalter für die Einstellungen gesprungen sind, wird der Fokus auf den ** OK ** Schalter im Dialog Ausführlichkeitsoptionen gesetzt.

## Meldungstypen

Das Listenfeld der zu sprechenden Elemente im Dialog Einstellungen listet die Informationstypen, die an- oder abgeschaltet werden können. Dieser Abschnitt enthält eine Liste der Informationstypen und eine Kurzbeschreibung jedes Elements.

## Position und Anzahl beim Navigieren in Listen ansagen

Beim Navigieren in einer Liste kann JAWS Ihre aktuelle Position innerhalb der Liste und die Gesamtanzahl der Listenelemente ansagen, wenn Sie sich zu einem Element mit **PFEIL RAUF ** oder ** PFEIL RUNTER ** bewegen. Zum Beispiel "Dateiname.txt, 5 von 9."

## Meldungen beim Anwendungsstart

Diese Option bezieht sich auf die Meldungen, die beim Start einer Anwendung gesprochen werden. Diese Informationen schließen oft Hinweise ein, wie man anwendungsabhängige Hilfe erhalten kann, manchmal sogar Hinweise, wie Sie die Anwendung einrichten können, damit sie besser mit JAWS läuft.

## Beschreibung des Steuerelements

Wenn für ein Steuerelement eine Beschreibung vorhanden ist, dann liest JAWS den beschreibenden Text, sobald das Element markiert ist.

## Gruppenfeldname

Elemente, die Gemeinsamkeiten haben, werden in Dialogfenstern oft in Gruppenfeldern zusammengefasst. So finden Sie beispielsweise in der Gruppe Alles Lesen in der Einstellungsverwaltung zwei Steuerelemente, die sich auf Alles Lesen beziehen. Das erste Element ist eine Gruppe von Auswahlschaltern, die festlegen, ob Dokumente zeilen-, satz- oder absatzweise vorgelesen werden. Das zweite ist das Kontrollfeld Leerzeilen sprechen. Da diese Elemente zusammengehören, wurden Sie im Gruppenfeld Dokumente vorlesen zusammengefasst. JAWS spricht die Worte "Dokumente vorlesen", wenn Sie sich zwischen diesen Elementen bewegen. Diese Informationen erscheinen auf dem Bildschirm, und Gruppenfelder sind in Windows weit verbreitet.

## Name des Steuerelements

Der Name des Steuerelements bezieht sich auf die Beschriftung eines bestimmten Steuerelements. Wenn Sie in einem Dialog navigieren, hören Sie zum Beispiel: "Vorname Eingabefeld Otto". In diesem Fall ist "Vorname" der Name des Steuerelements. Die Namen der Steuerelemente erscheinen auf dem Bildschirm und stellen normalerweise Informationen über dieses Steuerelement zur Verfügung.

## Steuerelementtyp

Der Steuerelementtyp bezeichnet den Typ eines bestimmten Steuerelements. Es gibt diverse allgemeine Typen von Steuerelementen in Windows. Diese beinhalten Eingabefelder, Ausklapplisten, Schalter, Listenfelder und so weiter. JAWS sagt normalerweise den Typ des Steuerelements nach dem Namen des Steuerelements und vor dem Inhalt des Steuerelements an. Wenn Sie in einem Dialog navigieren, hören Sie zum Beispiel: "Vorname Eingabefeld Otto". In diesem Fall ist der Steuerelementtyp "Eingabefeld", was bedeutet, dass es sich hierbei um ein Eingabefeld handelt, in das Sie Text eingeben können. Steuerelementtypen erscheinen nicht als Text auf dem Bildschirm.

## Dialogtitel

Dieser Typ bezieht sich auf den Titel eines Dialogfensters. Wenn Sie zum Beispiel im JAWS Fenster unter Stimmen den Punkt Allgemein... auswählen, spricht JAWS "Allgemeine Stimmeinstellungen Dialogfeld".

## Dialogtext

Dialogtext ist der Text in einem Dialog, der seine Funktion beschreibt. Dies kann ein einfacher Dialog mit einer Meldung, einem OK und einem Abbrechen Schalter sein. Dieser Typ bezieht sich weiterhin auf den beschreibenden Text eines mehrseitigen Dialogs.

## Dokumentname

Der Dokumentname bezieht sich auf Fenstertitel, die keine Dialogtitel sind. Dies kann der Titel eines Dokuments in Microsoft Word, ein Textdokument im Editor, oder der Titel einer E-Mail sein.

## Fehlermeldungen

Dieser Typ bezieht sich auf Fehlermeldungen, die JAWS spricht, wenn ein Vorgang nicht erfolgreich ausgeführt werden konnte. Wenn zum Beispiel jemand eine Tastenkombination drückt, um einen Schalter des Bildschirms auszuführen, der jedoch nicht sichtbar ist, spricht JAWS die Meldung "Schalter nicht gefunden". Diese Option bezieht sich nur auf JAWS Fehlermeldungen. Das Abschalten dieser Option bewirkt nicht, dass Ihnen keine Windows-Fehlermeldungen mehr vorgelesen werden.

## Hilfe-Sprechblasen

Sprechblasen werden von Windows genutzt, um mehr Informationen zu dem gewählten Element zu liefern. Bei aktiviertem Kontrollfeld für die Hilfe-Sprechblasen, liest JAWS automatisch die Informationen, sobald sie auf dem Bildschirm dargestellt werden.

## Hilfemeldungen

Hilfemeldungen sind Meldungen, die der Benutzer anfordert, und die JAWS aufgrund einer Tastenkombination spricht. Dies schließt alle Hilfe-Tastenkombinationen ein wie zum Beispiel **JAWS TASTE+F1 **, ** JAWS TASTE+W ** und ** JAWS TASTE+H **. Tastenkombinationen, die Informationen liefern, wie ** JAWS TASTE+F **, ** JAWS TASTE+5 **, ** ALT+ENTFERNEN ** usw. fallen auch in diese Kategorie.

**Hinweis:** Da diese Informationen immer beim Drücken einer Tastenkombination gesprochen werden, können sie nicht abgeschaltet werden.

## Elementnummer

Dieser Typ bezieht sich auf Meldungen, die die numerische Position eines Elements in einer Gruppe von Elementen angeben. Ein Beispiel hierfür ist, wenn JAWS "1 von 5" in einer Liste von Dateien oder einer Gruppe von Auswahlschaltern spricht.

## Elementstatus

Dieser Informationstyp wird verwendet, wenn JAWS den Status eines Steuerelements spricht. JAWS spricht zum Beispiel, wenn ein Element in einer Liste gewählt oder nicht gewählt ist, oder wenn ein Kontrollfeld aktiviert oder nicht aktiviert ist.

## JAWS Meldungen

Dies sind Meldungen, die Hilfestellungen geben, indem sie zum Beispiel mitteilen, dass ein Windows-Ereignis stattgefunden hat. So hilft zum Beispiel die Meldung "Fensteranfang" beim Navigieren, indem sie Ihnen mitteilt, dass Sie den oberen Fensterrand erreicht haben.

## Mausecho

JAWS kann Text sprechen, wenn Sie mit dem Mauszeiger darauf zeigen. Dieser Informationstyp steuert, ob lange oder kurze Meldungen zu hören sind, wenn die Maus über den Text bewegt wird, sofern die Option Mausecho in der Gruppe Mausecho der Einstellungsverwaltung aktiviert ist.

## Positionsinformation

Dieser Typ bezieht sich auf die Position eines Elements in Relation zur Gesamtstruktur, in der es sich befindet. So können zum Beispiel Zellkoordinaten einer Tabelle als Positionsinformation angesehen werden, weil sie die Lage dieser Zelle in der Tabelle beschreiben.

## Bildschirmmeldungen

Dieser Typ bezieht sich auf Text, der auf den Bildschirm geschrieben wird, der jedoch keinem bestimmten Element zugeordnet ist oder nicht automatisch von JAWS gelesen wird. So können zum Beispiel Auswahlschalter in Dialogen mit weiterführenden Informationen versehen sein, die sich in einem benachbarten Fenster befinden.

## Auswählen

Dieser Typ bezieht sich auf die Meldungen "Wähle" und "Auswahl aufheben", die gesprochen werden, wenn Sie Text auswählen.

## Gewähltes Element

Dieser Typ bezieht sich auf das automatische Vorlesen eines gewählten Elements in einem Dialogelement oder Fenster. Wenn der Fokus zum Beispiel auf einer Dateiliste landet, hören Sie "Dateien: Listenfeld, dokument1.doc nicht ausgewählt". In diesem Beispiel ist "dokument1.doc nicht ausgewählt" das gewählte Element.

## Intelligente Hilfe

Manchmal stellt JAWS Orientierungshilfen zur Verfügung, die nützlich sind, wenn Sie ein Programm neu erlernen, die aber nicht mehr nötig sind, sobald Sie mit dem Programm vertraut sind.

Eine Meldung, die Sie daran erinnert, **F6 ** zu drücken, um die Fensterausschnitte in einem Fenster der HTML Hilfe umzuschalten, kann zum Beispiel als eine solche betrachtet werden. Dieser Informationstyp kann als akustischer Tool Tipp angesehen werden.

## Status Informationen

Dieser Typ bezieht sich auf Meldungen, die Rückmeldungen über JAWS Befehle geben. Dazu gehört zum Beispiel die Meldung "Tastaturhilfe an", was den aktuellen Status der Tastaturhilfe wiedergibt. Das Abschalten dieser Option ist nicht empfehlenswert, weil dadurch wichtige gesprochene Informationen verlorengehen können.

## Popups

Diese Einstellung ermöglicht JAWS das automatische Ansagen von eingeblendeten Hinweisen in Windows.

## Sprache Auf Abruf

Sprache auf Abruf ist eine Funktion, die es ermöglicht, den Umfang der von JAWS beim Navigieren in Fenstern und Dokumenten oder beim Ausführen von anderen Aktionen über Kurztasten gesprochenen Informationen zu begrenzen. Standardmäßig sagt JAWS bei Nutzung von Sprache auf Abruf nur die Aktionen an, die einem eingeschränkten Satz an Kurztasten zugewiesen sind. Darüberhinaus stehen Mausecho und Tastaturecho weiterhin zu Verfügung, wodurch JAWS Text lesen kann, auf den Sie mit dem Mauszeiger zeigen und Zeichen und Wörter ansagt, während Sie Text eingeben. Sie können schnell zwischen Sprache auf Abruf und Voll-Sprache umschalten, indem Sie einen verschachtelten Befehl verwenden, oder Sie machen Sprache auf Abruf beim JAWS Start zum Standardlesemodus.

JAWS Anwender, die stark auf Braille angewiesen sind oder Fusion Anwender, die gut genug sehen können, um den Fokus auf dem Bildschirm zu verfolgen, können durch zuviel Sprachansagen beim Navigieren mit Tastatur und Maus schnell mit Informationen überflutet werden. Dennoch möchten Anwender weiterhin in der Lage sein, die Sprache zu verwenden, sobald Sie an einer Position auf dem Bildschirm angekommen sind, wo sie den Lesevorgang starten möchten, oder einfach einen Fenstertitel überprüfen möchten, oder die Spaltenüberschrift einer Tabelle oder eines Arbeitsblattes. oder einfach nur, um die Uhrzeit im Infobereich zu lesen. In beiden Situationen kann Sprache auf Abruf sehr hilfreich sein. Sie werden feststellen, dass Aktionen wie das Umschalten zwischen Anwendungen mit **ALT+TAB ** oder das Navigieren durch ein Dokument mit den ** PFEILTASTEN ** jetzt still vor sich gehen. Sprache wird nur noch verwendet, wenn ein eingeschränkter Satz von Lesebefehlen verwendet wird, wie etwa Sprechen des aktuellen Zeichens, des Worts, der Zeile, des Absatzes oder beim Ausführen von Alles Lesen. Sie haben auch die Möglichkeit, die Sprache vollständig abzuschalten.

Verschachtelte Befehle zum Umschalten von Sprache auf Abruf oder Sprache stummschalten:

- Drücken Sie **EINFÜGEN+LEERTASTE ** gefolgt von ** S **, um den Sprachmodus zwischen Voll-Sprache und entweder Sprache auf Abruf (Standard) oder Stumm umzuschalten.
- Solange der Sprachmodus Voll-Sprache nicht aktiv, können Sie **EINFÜGEN+LEERTASTE ** gefolgt von ** UMSCHALT+S ** ist drücken, um zwischen Sprache auf Abruf und Stumm umzuschalten.

Standardmäßig kehrt JAWS nach einem Neustart immer auf Voll-Sprache zurück, wenn diese Kurztasten verwendet wurden. Um die Änderungen an diesen Standard-Einstellungen dauerhaft zu ändern, gehen Sie wie folgt vor:

1. Drücken Sie **EINFÜGEN+F2 ** und wählen Sie Einstellungsverwaltung.
2. Wenn Sie möchten, dass Änderungen sich nur auf eine bestimmte Anwendung auswirken, wählen Sie diese in der Liste der Anwendungen. Ansonsten drücken Sie **STRG+UMSCHALT+D **, um die Standardeinstellungsdatei zu laden.
3. Erweitern Sie in der Strukturansicht die Gruppe Sprachoptionen.
4. Drücken Sie die **LEERTASTE ** auf Sprache auf Abruf, um die Standardauswahl zwischen Reduzierter Sprache oder Voll-Sprache zu ändern. Voll-Sprache ist standardmäßig aktiviert.
5. Drücken Sie die **LEERTASTE ** auf der Option Reduzierte Sprache, um die Standardauswahl zwischen Stummschalten und Sprache auf Abruf zu ändern. Sprache auf Abruf ist standardmäßig ausgewählt.
6. Wenn Sie keine von JAWS generierten Sound während aktiver Sprache Auf Abruf oder Stummschaltung hören möchten, drücken Sie **PFEIL RECHTS **, während Sie die Auswahlschaltergruppe für Reduzierte Sprache fokussieren und nutzen Sie dann die ** LEERTASTE **, um das Kontrollfeld JAWS Sounds abspielen zu deaktivieren. Dieses beinhaltet den hörbaren Ton, wenn Sie ** EINFÜGEN+LEERTASTE ** drücken, um eine verschachtelete Tastenkombination zu aktivieren, wie auch den Ton für die automatische Vervollständigung von Adressen in Outlook.
7. Erweitern Sie die Gruppe Sprache auf Abruf Einstellungen.
8. Wenn Sie nicht möchten, dass bei aktivem Sprache auf Abruf Ansagen durch Kurztasten für Lesebefehle erfolgen, dann verwenden Sie die **LEERTASTE **, um das Kontrollfeld Lesebefehle zulassen zu deaktivieren. Lesebefehle werden standardmäßig gesprochen.
9. Wenn das JAWS Mausecho aktiviert ist und Sie möchten Text hören, wenn Sie sich bei aktivem Sprache auf Abruf mit der Maus bewegen, dann nutzen Sie die **LEERTASTE **, um das Kontrollfeld Mausecho zulassen zu aktivieren. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.
10. Wenn das JAWS Tastaturecho auf Zeichen, auf Wörter oder auf Zeichen und Wörter gesetzt ist und Sie möchten hören, was Sie bei aktivem Sprache auf Abruf schreiben, dann verwenden Sie die **LEERTASTE **, um das Kontrollfeld Tastaturecho zulassen zu aktivieren. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.
11. Wählen Sie OK, um Ihre Änderungen zu speichern und die Einstellungsverwaltung zu schließen.

**Hinweis:** Wenn Sprache auf Abruf oder Sprache stummgeschaltet verwendet wird, dann sind auch alle von JAWS abgespielten Hinweistöne stumm. Dieses beinhaltet den hörbaren Ton, wenn Sie ** EINFÜGEN+LEERTASTE ** drücken, um eine Tastenebene zu aktivieren, wie auch den Ton für die automatische Vervollständigung von Adressen in Outlook.

## Der Dialog Sprachausgabeoptionen

# Gruppe Sprachausgaben

Sprachausgabeeinstellungen finden Sie in der Gruppe Sprachausgabe in der Einstellungsverwaltung. Mithilfe dieser Optionen werden WAV Dateien mit Softwaresynthesizern sanfter abgespielt.

## Wave-Dateien mit Software-Sprachausgaben erlauben

Ist dieses Kontrollfeld aktiviert, werden Wave-Dateien abgespielt, während die Sprachausgabe die Soundkarte verwendet, ohne dass der Sound manchmal stottert. Dieses Kontrollfeld steht nur in DEFAULT.JCF zur Verfügung und ist standardmäßig deaktiviert. Damit diese Änderung wirksam wird, muss JAWS nach dem Speichern der DEFAULT.JCF neu gestartet werden.

## Rücksetzfrequenz

Mit diesem Eingabefeld legen Sie fest, wie oft JAWS die Hardware-Kommunikation der Sprachausgabe zurücksetzt. So überprüft JAWS zum Beispiel bei Benutzung der Eloquence die Kommunikation mit der Soundkarte alle zwei Sekunden und stellt diese, wenn nötig, wieder her. Der Standardwert ist 2000 Millisekunden. Sollten Sie diesen Wert ändern wollen, empfehlen wir, dass Sie dies in einem Bereich von 1000 bis 3500 Millisekunden tun.

## Die Gruppe Textanalyse

Wenn ein Dokument nur unter Verwendung der Sprachausgabe bearbeitet wird, dann können Ungereimtheiten, wie unpassende Klammerungen, unbeabsichtigte Formatänderungen, zusätzliche Leerzeichen und verirrte Satzzeichen, öfter übersehen werden. Die Textanalyse ist eine Funktion, die Sie über derartige Fehler in Ihrem Dokument informiert, indem eine gesprochene Meldung oder eine WAV-Datei genutzt wird. Sie können JAWS so konfigurieren, dass auf diese Diskrepanzen über Anzahl, Beschreibung oder einen Sound während des Korrekturlesens über die Befehle Zeile lesen, Satz lesen, Absatz lesen, oder Alles Lesen hingewiesen wird. Brailleanwender werden von dieser Funktion profitieren, indem sie bereits einen Sound hören, bevor sie die gesamte Zeile auf Fehler überprüfen würden.

## Textanalyse einschalten

Die Textanalyse ist standardmäßig ausgeschaltet. Um sie einzuschalten, nutzen Sie eine der folgenden Möglichkeiten:

- Nutzen Sie die neue mehrschichtige Kurztaste **EINFÜGEN+LEERTASTE **, ** A **. Durch Drücken und Loslassen von ** EINFÜGEN+LEERTASTE **, und anschließendem Drücken von ** A **, können Sie die Textanalyse ein- oder ausschalten.
- Öffnen Sie die Einstellungsverwaltung (**EINFÜGEN+F2 **). Im Sucheingabefeld geben Sie "Textanalyse" ohne Anführungsstriche ein. Drücken Sie ** PFEIL RUNTER **, um zur Textanalyse in den gefilterten Ergebnissen zu springen. Drücken Sie ** EINGABE **, um eine Hinweisart auszuwählen.
- Nutzung der Schnelleinstellung (**EINFÜGEN+V **) Der Vorgang zur Auswahl einer Hinweisart ist identisch mit dem oben beschriebenen Vorgang für die Einstellungsverwaltung.

## Tastaturbefehle

Wurden mehrere Probleme im Dokument gefunden, drücken Sie **ALT+WINDOWS TASTE+I **, um zum nächsten Element zu springen, oder ** ALT+UMSCHALT+WINDOWS TASTE+I ** um zum vorherigen Element zu springen. Beachten Sie, dass diese Kurztasten weiterhin genutzt werden können, auch wenn die Textanalyse ausgeschaltet ist.

## Textanalyse Einstellungen

Nutzen Sie die Auswahlschalter dieser Gruppe, um die Art der Warnmeldung festzulegen, die Fehler beim Schreiben oder Korrekturlesen eines Dokuments (zeilenweise, satzweise, absatzweise, oder bei Alles Lesen) anzeigen soll. Wählen Sie Mit Sound anzeigen, um eine WAV Datei abzuspielen, wenn ein Formatierungsfehler erkannt wurde. Wählen Sie Anzahl ansagen, um die Anzahl der Ungereimtheiten im aktuellen Text anzusagen. Wählen Sie Ungereimtheit beschreiben, um eine Beschreibung des Fehlers zu erhalten. Die Standardeinstellung ist Aus.

## Die Gruppe Textverarbeitung

Nutzen Sie die Einstellungen dieser Gruppe, um festzulegen, wie JAWS Text liest. Dies beinhaltet, wie Großschreibung angesagt wird, wie Listenelemente, Zahlen und Daten angesagt werden, wie neue Zeilen angekündigt werden und vieles mehr.

## Allgemein

### Filter für sich wiederholende Zeichen

Mithilfe dieser Auswahlschalter legen Sie fest, wie oft ein Zeichen gesprochen wird, wenn JAWS auf eine aufeinanderfolgende Reihe desselben sich wiederholenden Zeichens stößt. Sie können aus den folgenden Optionen wählen: Drei, vier, fünf, sechs, Alle oder Zeichen zählen. Ist diese Option auf Drei, Vier, Fünf, Sechs oder Alle gesetzt, dann beachten Sie bitte, dass Leerzeichen nicht gemeldet werden. Leerzeichen werden jedoch gemeldet, wenn die Option Zeichen zählen eingestellt ist. Dies ist nützlich für Editoren, bei denen die Anzahl der Leerzeichen wichtig ist, z.B. beim Programmieren.

Standardmäßig werden die ersten drei sich wiederholenden Zeichen angesagt.

### Großschreibung ansagen

Mit dieser Option teilen Sie JAWS mit, wann Großschreibung angesagt werden soll. Standardmäßig meldet JAWS Großbuchstaben, wenn zeichenweise gelesen wird oder Wörter oder Zeilen buchstabiert werden.

### Listenelement

Legen Sie fest, wie Listenelemente beschrieben werden sollen:

- **Keines sprechen **: Diese Option schaltet die Beschreibung von Listenelementen ab.
- **Ausgewählte sprechen **: Ist diese Option gesetzt, sagt JAWS nur ausgewählte Listenelemente an.
- **Nicht ausgewählte sprechen **: Ist diese Option gesetzt, sagt JAWS nur ausgewählte Listenelemente an. Dieses ist die Standardeinstellung.
- **Beide sprechen **: Diese Option beschreibt Ihnen sowohl ausgewählte als auch nicht ausgewählte Listenelemente.

### Alphanumerische Daten buchstabieren

Legen Sie fest, wie JAWS alphanumerische Daten lesen soll:

- **Aus **: Dies ist die Standardeinstellung. Ist diese Option aktiviert, dann liest JAWS den Text so, wie er erscheint. So liest JAWS zum Beispiel die Zeichenfolge 123RON als einhundertdreiundzwanzig Ron.
- **Buchstabieren **: JAWS buchstabiert alphanumerische Daten. So liest JAWS zum Beispiel die Zahlenfolge 123RON als 1 2 3 R O N.
- **Phonetisch Buchstabieren **: JAWS verwendet das phonetische Alphabet, um alphanumerische Daten zu buchstabieren. So liest JAWS zum Beispiel die Zahlenfolge 123RON als 1 2 3 Richard Oskar Nordpol.

### Neue Zeilen ansagen, wenn

Im Sprach- und Soundmanager können Sie Text oder einen Sound festlegen, den Sie hören möchten, sobald Sie beim Eingeben oder Lesen auf eine neue Zeile gelangen. Aktivieren Sie ein oder mehrere der folgenden Kontrollfelder, um festzulegen, wann JAWS eine neue Zeile melden soll:

- **Editor bricht Text beim Schreiben um **: Editor beim Schreiben Text umbricht - JAWS meldet neue Zeilen, sobald Ihr Editor oder Ihre Textverarbeitung Text während der Eingabe auf die nächste Zeile umbricht.
- **User navigiert links oder recht über einen Zeilenumbruch hinaus **: JAWS sagt neue Zeilen an, wenn Sie wort- oder zeichenweise lesen.
- **Benutzer Text mit Zeilenumbruch-Zeichen liest **: JAWS meldet neue Zeilen, wenn Sie die JAWS Standard-Lesebefehle verwenden und auf die Steuerzeichen stoßen, die den Anfang einer neuen Zeile repräsentieren.
- **Während Alles lesen **: JAWS meldet, wenn es auf eine neue Zeile bei Alles lesen springt.

### Intelligente Worterkennung

In früheren Versionen hat JAWS beim Befehl Alles lesen nur das aktuelle Wort oder Zeichen gelesen, wenn der Cursor auf einer Texteinheit, wie zum Beispiel ein zusammengesetztes Wort, eine Telefonnummer, ein Datum, eine Uhrzeit oder eine mathematische Gleichung, positioniert war. Ist "Schlaues Lesen von Wörtern" aktiviert, dann liest JAWS die gesamte Texteinheit, sobald der Befehl Alles lesen gestartet wird.

**Hinweis:** Die Funktion "Schlaues Lesen von Wörtern" ist anwendungsabhängig. Wenn also Microsoft Word normalerweise die gesamte Texteinheit als ein Wort oder eine Zahl liest, dann macht JAWS das gleiche. Wenn eine Anwendung wie zum Beispiel WordPad dagegen jede einzelne Komponente der Texteinheit liest, dann tut JAWS das auch.

Wenn zum Beispiel in dem Satz "Er zieht hier gerade seine One-Man-Show ab" der Cursor irgendwo auf dem Wort "One" platziert ist, dann würde JAWS nur "One" lesen. Mit aktivierter Funktion "Schlaues Lesen von Wörtern" liest JAWS das gesamte zusammengesetzte Wort "One-Man-Show".

Ebenso würde JAWS in dem Satz "Die Besprechung ist auf 14:00 Uhr festgelegt" nur "Doppelpunkt" sagen, sollte sich der Cursor zwischen der 4 und dem Doppelpunkt befinden. Mit aktivierter Funktion "Schlaues Lesen von Wörtern" liest JAWS die gesamte Texteinheit als vierzehn Uhr.

Als letztes Beispiel schauen wir uns an, wie JAWS eine Telefonnummer liest. Ist der Cursor zwischen dem Gedankenstrich und der 444 in der Telefonnummer 1-800-444-4443 platziert, dann hat JAWS in früheren Versionen "444" gelesen. Mit aktivierter Funktion "Schlaues Lesen von Wörtern" liest JAWS die gesamte Telefonnummer "1-800-444-4443".

### Groß-/Kleinschreibung

Ist dieses Kontrollfeld aktiviert, werden Worte, die Großbuchstaben in der Mitte enthalten, wie zwei separate Wörter gesprochen. So wird der Skriptname SayAll als "Say all" gelesen, nicht als "Sayall". Dieses Kontrollfeld ist standardmäßig aktiviert.

### Wörterbuch verwenden

Ist dieses Kontrollfeld aktiviert, werden Worte, die gesprochen werden sollen, erst durch ein globales oder anwendungsabhängiges Wörterbuch verarbeitet. Für weitere Informationen finden Sie unter [Wörterbuchmanager](jdiction.chm::/jdiction/introduction_to_dictionary_manager.htm). Dieses Kontrollfeld ist standardmäßig aktiviert.

### Immer phonetisch buchstabieren

Ist dieses Kontrollfeld nicht aktiviert, dann bewirken die Befehle Aktuelles Wort oder Zeile buchstabieren (**EINFÜGEN+NUM 5 ** oder ** EINFÜGEN+PFPEIL RAUF ** zweimal schnell), dass JAWS jedes Zeichen wie erwartet spricht. Ist es aktiviert, dann hören Sie immer die phonetische Entsprechung eines jeden Zeichens im Wort oder der Zeile, wenn diese buchstabiert werden. Drücken Sie beispielweise ** EINFÜGEN+NUM 5 ** zweimal schnell, um "JAWS" zu buchstabieren, dann wird das Wort als Julius Anton Wilhelm Siegfried gelesen. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Pausieren beim zeichenweisen Navigieren, spricht das Zeichen phonetisch

Beim zeichenweisen Navigieren von Text kann es hilfreich sein, die phonetische Entsprechung für einen bestimmten Buchstaben zu hören. Ist diese Option aktiviert, dann bewirkt ein kurzes Pausieren auf einem Buchstaben, dass JAWS dieses phonetisch ansagt.

JAWS bietet auch eine manuelle Option, um fortlaufend phonetische Ansagen beim zeichenweisen Navigieren ansagen zu lassen. Die Kurztaste für Zeichen Lesen zweimal schnell gedrückt, schaltet diese Funktion ein und aus. Wenn Sie die manuelle Option für alle Buchstaben verwenden, dann wird die pausierte Ansage nicht durchgeführt.

Standardmäßig wird das phonetische Ansagen ausgeführt, wenn Sie für eineinhalb Sekunden beim zeichenweisen Navigieren pausieren.

### Aktualisierungen der Live Regionen ansagen

Diese Option legt fest, ob Aktualisierungen der Live Regionen auf Webseiten, die dieses unterstützen, angesagt werden sollen. Ist diese Einstellung deaktiviert, dann werden keine Aktualisierungen der Live Regionen gesprochen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Sprache der Tastatur erkennen

Ist das Kontrollfeld aktiviert, dann verwendet JAWS die geeignete Stimmensprache, die mit der aktuell aktiven Tastatursprache übereinstimmt. Wenn Sie beispielsweise die Eingabesprache auf Spanish umschalten, dann wird JAWS, sofern verfügbar, automatisch auf eine spanische Stimme umschalten, sobald Sie Text eingeben.

Deaktivieren Sie dieses Kontrollfeld, wenn Sie möchten, dass JAWS die aktuelle Stimmensprache weiterhin verwendet, auch wenn Sie die Tastatursprache auf eine andere Sprache umschalten.

Dieses Kontrollfeld ist standardmäßig aktiviert.

### Erweiterte Eingabeunterstützung

Wenn Sie dieses Kontrollfeld aktivieren, liefert JAWS eine erweiterte Unterstützung für Eingabe- und Rich-Edit-Bereiche. Diese Funktion sorgt für eine höhere Genauigkeit, wenn Text in diesen Feldern gelesen wird. Zusätzlich kann JAWS eine größere Auswahl von Textattributen und Sprachen besser automatisch erkennen. Es kann jedoch sein, dass dadurch die Reaktionszeit in bestimmten Programmen verlangsamt wird. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Sprich Komponentenstatus zuerst

Ist dieses Kontrollfeld aktiviert, wird der Status des Listenelements vor dem Namen gesprochen. Ist es deaktiviert, wird der Status des Listenelements nach dem Namen gesprochen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Sprich Fenstertyp zuerst

Ist dieses Kontrollfeld aktiviert, wird der Fenstertyp vor dem Fenstertitel oder Text gesprochen. Ein Beispiel für einen Fenstertyp ist "Dialogfeld". Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Sprich Fensterstatus zuerst

Ist dieses Kontrollfeld aktiviert, wird der Fensterstatus vor dem Fenstertitel oder Text gesprochen. In diesem Fall würde JAWS "Nicht aktiviert Kontrollfeld Groß-/Kleinschreibung" melden und nicht "Kontrollfeld Groß-/Kleinschreibung nicht aktiviert". Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Eingerückte Zeichen sprechen

Wenn dieses Kontrollfeld aktiviert ist, werden Veränderungen des Einzugs angesagt, entsprechend der Regeln, die im Sprach- und Soundmanager festgelegt wurden. Im Sprach- und Soundmanager legen Sie fest, wie JAWS Einzüge melden soll, aber dieses Kontrollfeld gibt an, ob Einzüge ein- oder ausgeschaltet sind. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Zahlen- und Datumsverarbeitung

### Numerische Datumsverarbeitung

Legen Sie fest, wie JAWS Daten in numerischen Formaten lesen soll:

- **Keine Übersetzung **: Ist diese Option gewählt, werden numerische Datumsangaben als Zahlen gelesen. Diese Option ist standardmäßig ausgewählt.
- **Einige Übersetzung **: Ist diese Option gesetzt, werden numerische Datumsangaben im Format Monat, Tag, Jahr, getrennt durch Schräg- oder Bindestriche, übersetzt. So wird 03-16-05 als "16. März 2005" gelesen.
- **Erweiterte Übersetzung **: Ist diese Option gewählt, werden zusätzlich zu der Übersetzung bei Einige Übersetzungen auch Datumsangaben im Format Monat, Tag, getrennt durch Schräg- oder Bindestriche, übersetzt. So wird 03-16 als "16. März" gelesen.

Wenn Sie Erweiterte Übersetzung wählen, werden viele Brüche als Datum gelesen. So wird 1/3 als "January 3rd" gelesen.

Die JAWS Datumsverarbeitung berücksichtigt nun auch die regionalen Windows Einstellungen der Datums-Trennungszeichen, um zu ermitteln, was als Datum behandelt werden soll. So wird zum Beispiel aus "17/8/2002" "17 August 2002", wenn die regionalen Einstellungen auf Australisches Englisch gesetzt sind, aber es wird nicht als Datum erkannt, wenn Amerikanisches Englisch eingestellt ist. Das Datum "3/2/2002" wird als "March 2, 2002" erkannt, wenn die regionalen Einstellungen auf Amerikanisches Englisch gesetzt sind, aber wenn Australisches Englisch eingestellt ist, wird es als "February 3, 2002" erkannt.

**Hinweis:** Wenn der Gedankenstrich als Datums-Trennzeichen in den regionalen Windows Einstellungen definiert ist, wird er nicht als Datumstrenner identifiziert.

### Systemgebietsschema für das Datumsformat verwenden

Wenn diese Option ausgewählt ist, verwendet JAWS das aktuelle Benutzer- oder Systemgebietsschema, um Informationen über das Format von Datumsangaben zu erhalten. Deaktivieren Sie dieses Kontrollfeld, damit JAWS die Informationen zum Datumsformat entweder über die Sprache des Dokuments oder von der Sprachausgabe erhält. Dieses Kontrollfeld wird in Verbindung mit der Ausklappliste Numerische Datumsverarbeitung verwendet und ist standardmäßig aktiviert.

### Zahlenbearbeitung

Legen Sie fest, wie Zahlen gelesen werden sollen:

- **Von Sprachausgabe kontrolliert **: Ist diese Option gewählt, legt die Sprachausgabe fest, wie Zahlen gelesen werden. Dieses ist die Standardeinstellung.
- **Von JAWS kontrolliert **: Wenn diese Option ausgewählt ist, dann gibt JAWS die Zahlen so an die Sprachausgabe weiter, wie sie gelesen werden sollen. Verwenden Sie diese Einstellung, wenn die aktuell verwendete Sprachausgabe bei der Verwendung der Einstellung ** Von Sprachausgabe gesteuert ** die Zahlen nicht korrekt spricht.
- **Einzelne Ziffern **: Ist diese Option gewählt, werden Zahlen als einzelne Ziffern gelesen. So wird zum Beispiel 123 als "eins zwei drei" gelesen.
- **Paare **: Ist diese Option gewählt, werden Zahlen als Folge von Ziffernpaaren gelesen. So wird zum Beispiel 1050 als "zehn fünfzig" gelesen. Wenn sich die Zahl aus einer ungeraden Anzahl von Ziffern zusammensetzt, wird die erste Ziffer einzeln gelesen und der Rest paarweise. So wird zum Beispiel 123 als "Eins dreiundzwanzig" gelesen.
- **Ganze Zahlen **: Ist diese Option gewählt, werden Zahlen immer als ganze Zahlen gelesen. So wird zum Beispiel 123 als "einhundertdreiundzwanzig" gelesen.

### Währungserkennung aktivieren

Ist diese Einstellung ausgewählt, dann erkennt JAWS Währungszeichenfolgen wie „$100“ und konvertiert sie in Text wie „einhundert US-Dollar“. Deaktivieren Sie dieses Kontrollfeld, um die Erkennung von Währungszeichenfolgen zu deaktivieren. Dieses Kontrollfeld ist verfügbar, wenn die Ausklappliste Zahlenverarbeitung auf **Von JAWS gesteuert ** festgelegt ist und ist standardmäßig aktiviert.

### Einzelne Ziffern sprechen, wenn die Zahl enthält:

**Hinweis:** Dieses Steuerelement wird zusammen mit der Ausklappliste Zahlenbearbeitung verwendet. Das heißt, dass JAWS beim Lesen eines Zahlenstrings zuerst den Wert aus der Ausklappliste Zahlenbearbeitung heranzieht, dann die Ausklappliste Einzelne Ziffern sprechen, wenn die Zahl enthält.

Legen Sie in dieser Ausklappliste einen Schwellenwert für JAWS fest, so dass ein Zahlenstring als einzelne Ziffern gelesen wird. Enthält ein String zum Beispiel fünf oder mehr Ziffern, dann liest JAWS jede Ziffer als einzelne Zahl. Der String 54321 wird 5 4 3 2 1 gelesen.

Dies ist besonders hilfreich, wenn es sich um lange Zahlenstrings wie zum Beispiel Telefonnummern und Seriennummern handelt. Bitte beachten Sie, dass diese Regel unter den folgenden Bedingungen angewendet wird: Dem Zahlenstring enthält kein Alpha-Zeichen am Anfang oder am Ende, und er enthält auch keinen Bindestrich, Doppelpunkt, Dezimalkomma oder Schrägstrich, um die Ziffern zu trennen, ähnlich wie bei Telefonnummern, Zeit- oder Datums-Formatierung. Wenn Sie möchten, dass der Bindestrich in einem Zahlenstring ignoriert wird, dann müssen Sie das Kontrollfeld Bindestriche aktivieren, das sich unter dieser Ausklappliste befindet.

Wählen Sie einen der folgenden Einträge in dieser Ausklappliste:

- **Fünf oder mehr Ziffern ** (Standard)
- **Sechs oder mehr Ziffern **
- **Sieben oder mehr Ziffern **
- **Acht oder mehr Ziffern **
- **Von Sprachausgabe kontrolliert **: Ist dieser Eintrag ausgewählt, dann legt die Sprachausgabe fest, wie Zahlen gelesen werden.

### Wenn Zahl Bindestriche enthält

**Hinweis:** Dieses Steuerelement wird genau wie die Ausklappliste "Einzelne Ziffern sprechen, wenn die Zahl enthält" in Zusammenhang mit der Ausklappliste Zahlenbearbeitung verwendet. Das heißt, dass JAWS die Zahlen aufgrund des Wertes in der Ausklappliste Zahlenbearbeitung verarbeitet. Ist dieses Kontrollfeld aktiviert, dann werden Zahlen, die Bindestriche ohne Leerzeichen enthalten, als einzelne Ziffern gelesen.

Aktivieren Sie dieses Kontrollfeld, damit Zahlenstrings, die Bindestriche enthalten, von JAWS als einzelne Ziffern gelesen werden. Diese Regel wird angewendet, wenn der Bindestrich vor oder hinter einer Ziffer steht, ähnlich wie bei Telefonnummern-Formatierung. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Dollars sprechen

Ist dieses Kontrollfeld aktiviert, werden Zahlenausdrücke, denen ein Dollarzeichen $ voransteht, als Dollar und Cent gelesen. So wird $9.95 als "neun Dollar und fünfundneunzig Cents" gelesen. Damit Beträge unter einem Dollar richtig gelesen werden, muss die 0 im Betrag vorhanden sein. So wird $0.02 als "zwei Cents" gelesen. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Korrektur

### Rechtschreibung beim Navigieren erkennen

Nutzen Sie diese Ausklappliste, um festzulegen, wie auf Rechtschreibfehler beim Lesen eines Dokuments hingewiesen wird. Die folgenden Optionen stehen zur Verfügung:

- **Aus ** - JAWS wird nicht auf Rechtschreibfehler hinweisen.
- **Sound abspielen ** - Ein Signalton ist hörbar, wenn Sie auf ein falsch geschriebenes Wort navigieren.
- **Schreibfehler ansagen ** - JAWS sagt "Schreibfehler" wenn Sie auf ein falsch geschriebenes Wort navigieren. Dies ist die Standardeinstellung.

### Rechtschreibung beim Schreiben erkennen

Über dieses Kontrollfeld können Sie auswählen, wie beim Schreiben auf Rechtschreibfehler hingewiesen wird Ist es deaktiviert, dann wird auf keine Schreibfehler hingewiesen. Ist es aktiviert, dann wird ein Sound abgespielt, sobald die Anwendung feststellt, dass Sie ein Wort entweder falsch geschrieben oder korrigiert haben. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Anzahl der Rechtschreibfehler ansagen

Aktivieren Sie dieses Kontrollfeld, um zuerst die Anzahl der Rechtschreibfehler zu hören, bevor beim zeilenweisen Lesen der Text der Zeile angesagt wird. Dieses Kontrollfeld ist standardmäßig nicht aktiviert und steht nicht zu Verfügung, wenn das Erkennen der Rechtschreibung beim Navigieren ausgeschaltet ist.

### Grammatik und erweiterte Rechtschreibprüfung

Ist dieses Kontrollfeld aktiviert, dann spricht JAWS eine Meldung, um darauf hinzuweisen, wenn Sie beim Navigieren auf einen Grammatik- oder erweiterten Schreibfehler treffen oder diesen verlassen. Dieses Kontrollfeld ist für alle Anwendungen standardmäßig aktiviert, damit JAWS in webbasierten Textverarbeitungsprogrammen wie Google Docs erkannte Grammtikfehler ansagt. Dieses Kontrollfeld ist standardmäßig nur in Microsoft Word und Outlook deaktiviert, da die Grammatik- und Rechtschreibprüfung in Word viel strenger ist, als in den meisten Online-Textverarbeitungsprogrammen, was zu weit umfangreicheren Ansagen führen kann.

### Anzahl der Grammatik/Rechtschreibfehler ansagen

Aktivieren Sie dieses Kontrollfeld, um zuerst die Anzahl der Grammatik- oder erweiterten Schreibfehler zu hören, bevor beim zeilenweisen Lesen der Text der Zeile angesagt wird. Dieses Kontrollfeld ist standardmäßig nicht aktiviert und steht nicht zu Verfügung, wenn das Erkennen von Grammatik- und erweiterten Schreibfehlern beim Navigieren ausgeschaltet ist.

### Optionen auf Alles Lesen anwenden

Aktivieren Sie dieses Kontrollfeld, damit JAWS Rechtschreib- und Grammatikfehler beim Lesen eines Dokuments während Alles Lesen (**EINFÜGEN+PFEIL RUNTER **) ansagt. Dieses Kontrollfeld ist standardmäßig nicht aktiviert und steht nicht zu Verfügung, wenn sowohl Rechtschreib- als auch Grammatikfehlererkennung ausgeschaltet ist.

### Kommentar ansagen

Verwenden Sie diese Ausklappliste, um festzulegen, wie JAWS Sie darauf hinweist, wenn Sie auf Text mit Kommentaren in webbasierten Editoren wie Google Docs stoßen. Zur Auswahl stehen **Aus **, ** Sprechen ** oder ** Sound abspielen **. Standardmäßig spricht JAWS eine Nachricht, wenn Sie in einen kommentierten Text hinein- oder aus einem kommentierten Text heraus navigieren.

### Durchgestrichen Ansage

Verwenden Sie dieses Kombinationsfeld, um auszuwählen, wie JAWS die Durchstreichungsformatierung in webbasierten Editoren wie Google Docs und auf Webseiten anzeigt. Ein Durchstreichen ist eine horizontale Linie, die durch Text gezogen wird, um Inhalte, die gelöscht, geändert oder nicht mehr relevant sind, visuell zu kennzeichnen, ohne sie vollständig aus dem Dokument zu entfernen. Zur Auswahl stehen Aus, Sprechen oder Sound abspielen. Standardmäßig spricht JAWS eine Nachricht, wenn Sie in einen durchgestrichenen Text hinein- oder aus einem durchgestrichenen Text heraus navigieren.

### Eingefügten Text ansagen

Wenn aktiviert, dann meldet JAWS, wenn Sie beim Navigieren in webbasierten Textverarbeitungsprogrammen wie Google Docs auf eingefügten Text stoßen oder diesen wieder verlassen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Gelöschten Text ansagen

Wenn aktiviert, dann meldet JAWS, wenn Sie beim Navigieren in webbasierten Textverarbeitungsprogrammen wie Google Docs auf gelöschten Text stoßen oder diesen wieder verlassen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Markierten Text ansagen

Wenn aktiviert, dann meldet JAWS, wenn Sie beim Navigieren in webbasierten Textverarbeitungsprogrammen wie Google Docs auf Text innerhalb eines markierten Elements stoßen oder diesen wieder verlassen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Ansage von vorgeschlagenem Text

Wenn aktiviert, dann meldet JAWS, wenn Sie beim Navigieren in webbasierten Textverarbeitungsprogrammen wie Google Docs auf vorgeschlagenen Text stoßen oder diesen wieder verlassen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Rechtschreibfehler unterstreichen

Aktivieren Sie dieses Kontrollfeld, um Schreibfehler auf der Braillezeile mit den Punkten 7 und 8 anzuzeigen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Grammatik und erweiterte Rechtschreibfehler unterstreichen

Aktivieren Sie dieses Kontrollfeld, um Grammatik- und erweiterte Schreibfehler auf der Braillezeile mit den Punkten 7 und 8 anzuzeigen. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

### Steuerzeichen wie etwa Seitenumbrüche filtern

Deaktivieren Sie dieses Kontrollfeld, um die Brailleausgabe für verschiedene Steuerzeichen, wie etwa Seitenumbrüche, zu erlauben. In Abhängigkeit von der verwendeten Anwendung können verschiedene Steuerzeichen angezeigt werden. Diese werden sowohl für Computerbraille, als auch in der Braille Kurzschriftausgabe angezeigt. Dieses Kontrollfeld ist standardmäßig aktiviert.

## Sprachenverarbeitung

### Sprache erkennen

Diese Gruppe von Auswahlschaltern ermöglicht das Konfigurieren der automatischen Spracherkennung. Diese Funktion funktioniert mit Sprachausgaben, die verschiedene Sprachen unterstützen, wie zum Beispiel der Eloquence oder Vocalizer Expressive. Bei Vocalizer Expressive müssen Sie Stimmen für bestimmte Sprachen installieren, damit die Spracherkennung ordnungsgemäß ausgeführt werden kann.

Wählen Sie **Basierend auf Dokument- oder Webseiten-Markierung **, damit JAWS Änderungen der Sprache in Dokumenten erkennt, die Sprachtags unterstützen. JAWS liest dann mit der entsprechenden Sprachausgabestimme für die angegebene Sprache. Findet JAWS zum Beispiel spanische Tags auf einer Webseite, dann wird der Text mit einer spanischen Stimme gelesen.

Wählen Sie **Basierend auf Text (Nur Vocalizer Expressive)**, damit JAWS die Sprache anhand der im Dokument verwendeten Textzeichen erkennt und mit der entsprechenden Vocalizer Expressive-Stimme vorliest. Wenn JAWS zum Beispiel arabische Zeichen erkennt, schaltet JAWS zum Lesen auf eine arabische Stimme um, sofern verfügbar.

Wählen Sie **Basierend auf KI-Vorhersagen **, damit JAWS einen KI-Algorithmus verwendet, der auf Ihrem Computer ausgeführt wird, um die Sprache anhand einer kleinen Textstelle zu erkennen, bevor dieser vorgelesen wird. JAWS schaltet dann auf die entsprechende Stimme für diese Sprache um, falls vorhanden.

Wählen Sie **Aus **, wenn Sie nicht möchten, dass JAWS die Sprachausgabestimme wechselt, wenn Text vorgelesen wird, der in einer anderen Sprache geschrieben wurde.

### Wenn Spracherkennung auf Text basiert

Diese Gruppe von Auswahlschaltern ermöglicht zu bestimmen, wie Zahlen gesprochen werden, wenn die Spracherkennung auf ”Basierend auf Text” festgelegt ist. Sie können wählen, ob Zahlen mit der Primärstimme angesagt werden sollen, oder ob Sie Zahlen mit einer Stimme lesen lassen möchten, die dem Sprachkontext der Zahlen entspricht.

Die Standardeinstellung ist, dass die Zahlen mit der Primärstimme vorgelesen werden.

### Erweiterte Unterstützung von Rechts-nach-Links-Sprachen

Aktivieren Sie dieses Kontrollfeld, um sicherzustellen, dass Text, der von Recht nach Links geschrieben ist, von der Sprachausgabe in der richtigen Reihenfolge gelesen wird.

Dieses Kontrollfeld ist standardmäßig in allen Sprachen nicht aktiviert, außer für Arabisch und Hebräisch.

### Dialekte verallgemeinern

Dieses Kontrollfeld legt fest, ob JAWS beim Umschalten auf eine andere Sprache zu einem bestimmten Dialekt dieser Sprache umschalten soll. Wenn diese Option aktiviert ist und JAWS zum Beispiel eine englischsprachige Webseite liest, die Amerikanisches, Britisches und Australisches Englisch enthält, dann schaltet JAWS nicht auf eine andere Sprache um, da die Basis all dieser Dialekte Englisch ist. Sollte JAWS jedoch auf eine andere Sprache, wie zum Beispiel Deutsch oder Französisch, treffen, dann wird auf diese Sprache umgeschaltet.

Dieses Kontrollfeld ist standardmäßig aktiviert und ist nur verfügbar, wenn die Spracherkennung auf “Basierend auf Dokument- oder Webseiten-Markierung” gesetzt ist.

## Die Gruppe Benutzereinstellungen

Nutzen Sie die Einstellungen dieser Gruppe, um Tastatureingaben und das Bildschirmecho zu steuern. Erfahrene Anwender mögen es bevorzugen, wenn das Tastaturecho ausgeschaltet oder auf nur Wörter eingestellt ist, während unerfahrene Anwender es möglicherweise hilfreich finden, wenn jeder Buchstabe beim Schreiben gesprochen wird. Tastaturecho und Unterbrechen beim Schreiben sind Beispiele für Tastatureingaben. Ein Beispiel für Bildschirmecho wäre die Ansage des Fortschrittsbalkens.

## Ansage der Fortschrittsbalkenaktualisierung

Programme verwenden häufig Fortschrittsbalken, um den Status eines Vorgangs anzuzeigen, wie zum Beispiel bei einer Installation. Wählen Sie in dieser Liste eine Zeitspanne aus, um festzulegen wie oft JAWS den Status von Fortschrittsbalken während eines Prozesses ansagen soll. Wählen Sie "Alles ansagen", wenn JAWS den Status ansagen soll, sobald er sich verändert. Wählen Sie "Aus", wenn JAWS den Status nicht melden soll.

**Tipp:** Sie können sich den Namen und den Status eines Fortschrittsbalkens jederzeit mit **JAWS TASTE+TAB ** ansagen lassen.

## Tastaturecho

Mit diesen Auswahlschaltern legen Sie fest, wie Informationen, die Sie eingeben, gesprochen werden. Standardmäßig wird jedes Zeichen, das Sie eingeben, gesprochen. Wählen Sie Ihre gewünschte Einstellung durch einen der vier Auswahlschalter:

- **Aus **: Wählen Sie diese Option, wenn Sie keine Rückmeldung beim Schreiben wünschen.
- **Zeichen **: Jedes Zeichen, das Sie eingeben, wird gesprochen.
- **Wörter **: Jedes Wort wird gesprochen, nachdem Sie ** LEERTASTE ** oder ** EINGABE ** drücken.
- **Zeichen und Wörter **: Beim Eingeben werden Zeichen gesprochen, ganze Wörter werden gesprochen, wenn Sie ** LEERTASTE ** oder ** EINGABE ** drücken.

## Bildschirmecho

Diese Auswahlschalter erlauben das Einstellen, welche Informationen gesprochen werden, wenn sie sich am Bildschirm ändern.

- **Keinen Text sprechen **: Änderungen am angezeigten Text werden nicht gesprochen.
- **Hervorgehobenen Text sprechen **: Änderungen an hervorgehobenem Text werden gesprochen.
- **Gesamten Text sprechen **: Es werden Änderungen am gesamten Text gesprochen.

## Unterbrechen beim Schreiben

Ist dieses Kontrollfeld aktiviert, wird die Sprache beim Eingeben von Zeichen und Ziffern unterbrochen. Diese Option ist standardmäßig aktiviert.

## Modus der Einfügetaste

Ist dieses Kontrollfeld aktiviert, funktioniert die **EINFÜGETASTE ** als Feststelltaste (sticky key). Jede Taste, die direkt nach der ** EINFÜGETASTE ** gedrückt wird, wird so behandelt, als würde sie mit der ** EINFÜGETASTE ** zusammen gedrückt. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Tastaturwiederholung

Ist dieses Kontrollfeld aktiviert, wird eine Taste, die festgehalten wird, wiederholt gedrückt. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Virtuellen Betrachter am Bildschirm anzeigen

Mit diesem Kontrollfeld legen Sie fest, dass der virtuelle Betrachter bei Drücken von **JAWS TASTE+F1 **, oder bei anderen wichtigen Mitteilungen auf dem Bildschirm sichtbar dargestellt wird. In Trainingsumgebungen oder bei Schulungen kann ein Anzeigen des virtuellen Betrachters sehr hilfreich sein. Allerdings kann sich die Anzeige des Betrachters in Präsentationsanwendungen, wie PowerPoint, auch störend auswirken. Die Funktionalität von JAWS ist jedenfalls unabhängig davon, ob der virtuelle Betrachter sichtbar ist oder nicht.

## Virtuellen Cursor für Windows Universal Apps aktivieren

Ist dieses Kontrollfeld aktiviert ist, dann wird der virtuelle Cursor zum Lesen in Windows Universal Apps verwendet. Bei Verwendung des virtuellen PC-Cursors werden Windows Universal Apps ähnlich wie Webseiten dargestellt, sodass Sie mit diesen Apps so interagieren können, als würden Sie in einem Browser auf einer Webseite navigieren. Windows Universal Apps werden üblicherweise über den Microsoft Store bezogen, einige sind jedoch als Teil von Windows bereits vorinstalliert

Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

## Lautstärke anderer Programme verringern, während JAWS spricht

Ist dieses Kontrollfeld aktiviert, dann wird während JAWS spricht, die Lautstärke von anderen Programmen reduziert. Dieser wird weitläufig als Audio Ducking bezeichnet. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

Sie können auch **EINFÜGEN+LEERTASTE **, gefolgt von ** D ** drücken, um diese Funktion umzuschalten. Wenn die Reduzierung anderer Audiosignale standardmäßig deaktiviert ist, dann wird es daraufhin eingeschaltet, wenn es standardmäßig eingeschaltet ist, dann wird es ausgeschaltet. Die aktuelle Einstellung bleibt solange aktiv, bis Sie die verschachtelte Kurztaste erneut drücken oder bis Sie JAWS neu starten.

## Bei bestimmten Bluetooth Kopfhörern oder Soundkarten das Abbrechen der Sprache verhindern

Viele moderne Bluetooth Kopfhörer und Lautsprecher, wie auch einige Onboard-Soundkarten gehen in den Standby Modus, um Akkuleistung zu sparen, wenn Sie für eine bestimmte Zeit keinen Sound erhalten haben. Wenn dieses Kontrollfeld aktiviert ist, dann hält JAWS diese aktiv, indem eine konstante Stille abgespielt wird. Sie werden dabei nichts hören, aber Ihr Gerät verbleibt im aktiven Status, so dass Sprache unverzögert wiedergegeben werden kann. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

**Hinweis:** Der Akkustand Ihrer Kopfhörer/Lautsprecher kann sich schneller verringern, wenn Sie diese Einstellung aktivieren.

## Benachrichtigungsregeln aktivieren

Der JAWS Benachrichtigungsverlauf, **EINFÜGEN+LEERTASTE ** gefolgt von ** N **, zeigt einen Verlauf der Benachrichtigungen, die Sie in den vergangenen 24 Stunden von Windows und Anwendungen erhalten haben, während JAWS oder Fusion ausgeführt wurde. Mit diesem Hilfsprogramm können Sie ganz einfach Regeln für die verschiedenen Benachrichtigungen definieren, um festzulegen, wie viel oder wie wenig Sie in Zukunft für jede Benachrichtigung sowohl in Sprache als auch in Braille erhalten möchten. So können Sie beispielsweise festlegen, dass JAWS Benachrichtigungen von Outlook nur dann anzeigt, wenn diese einen bestimmten Text enthalten, oder dass JAWS nur einen bestimmten Ton abspielt, anstatt Sie mit einer gesprochenen Nachricht zu unterbrechen.

Deaktivieren Sie dieses Kontrollfeld, um alle vom Anwender erstellten Benachrichtigungsregeln auszuschalten. Nach der Deaktivierung können Sie keine Regeln mehr für die ausgewählte Benachrichtigung in der Liste erstellen, und JAWS oder Fusion werden alle Benachrichtigungen ankündigen, sobald sie empfangen werden.

Dieses Kontrollfeld ist standardmäßig aktiviert.

## Die Gruppe Visuelle Verfolgung

## Virtuelles Menüband hervorheben

Wenn das virtuelle Menüband aktiv ist, dann können Sie dieses Kontrollfeld aktivieren, damit JAWS jedes Element beim Navigieren hervorhebt, um auch visuell darzustellen, welches Element aktuell gewählt ist. Diese Option ist standardmäßig ausgewählt. ## Virtuellen PC Cursor hervorheben

Wird in Webseiten, PDF Dokumenten oder E-Mail-Nachrichten mit dem virtuellen PC Cursor navigiert, aktivieren Sie dieses Kontrollfeld, damit JAWS den aktuell gewählten Text hervorhebt, wenn dieser gesprochen wird. Hierdurch wird ermöglicht, dass Sie dem Text auch visuell leicht folgen können, während JAWS liest. Diese Einstellung ist standardmäßig aktiviert in den Programmen Edge, Chrome, Firefox, Adobe Reader und Microsoft Mail. Drücken Sie **F6 **, um in den Bereich Visuelle Verfolgung zu springen, um Attribute der Hervorhebung, inklusive Farbe, Formatierung (Rechteck, Block oder Unterstrichen), Transparenz, Breite und Abstand festzulegen.

## Touch Cursor hervorheben

Wird mit dem Touch Cursor oder mit Touch Gesten navigiert, aktivieren Sie dieses Kontrollfeld, damit JAWS das aktuell gewählte Element hervorhebt, um auch visuell die Position des Touch Cursors anzuzeigen. Drücken Sie **F6 **, um in den Bereich Visuelle Verfolgung zu springen, um Attribute der Hervorhebung, inklusive Farbe, Formatierung (Rechteck, Block oder Unterstrichen), Transparenz, Breite und Abstand festzulegen.

## Visuelle Einstellungen

### Farbe

Wählen Sie die Farbe, die für die visuelle Hervorhebung genutzt wird. Die Standardfarbe ist grün.

### Typ

Wählen Sie einen der folgenden Stile, der für die visuelle Hervorhebung genutzt werden soll:

- **Rechteck ** - Platziert eine rechteckige Umrandung um den Text, wenn dieser gesprochen wird.
- **Block ** - Nutzt ein ausgefülltes Rechteck, um den Text hervorzuheben, wenn dieser gesprochen wird.
- **Unterstrichen ** - Unterstreicht den Text, wenn dieser gesprochen wird.

Der Standardstil ist Block.

### Transparenz

Verwenden Sie den Schieberegler, um die Deckkraft für die visuelle Hervorhebung um den gewählten Text herum zu erhöhen oder zu verringern. Höhere Stufen der Transparenz ermöglichen das Sehen von Text und Bildern unter der Hervorhebung. Drücken Sie **F6 **, für den Zugriff auf den Schieberegler und dann die ** Pfeiltasten **, zum Ändern des Wertes.

### Breite

Verwenden Sie den Schieberegler, um die Breite der Umrandung für die Stile Rechteck oder Unterstrichen zu erhöhen oder zu verringern. Breitere Umrandungen sind einfacher zu sehen und heben sich mehr ab. Dünnere Umrandungen sind zurückhaltender und lassen mehr Text sichtbar. Drücken Sie **F6 **, für den Zugriff auf den Schieberegler und dann die ** Pfeiltasten **, zum Ändern des Wertes.

### Abstand

Verwenden Sie den Schieberegler, um die Größe des weißen Zwischenraums zwischen gewähltem Text und der Umrandung der Hervorhebung für die Stile Rechteck, Block oder Unterstrichen zu erhöhen oder zu verringern. Drücken Sie **F6 **, für den Zugriff auf den Schieberegler und dann die ** Pfeiltasten **, zum Ändern des Wertes.

## Stimmenaliase

Mithilfe von Stimmenaliasen können Sie nun eine beliebige Kombination von Sprecher, Stimmhöhe und Geschwindigkeit unter einem Stimmennamen abspeichern, an den JAWS sich erinnert. Danach können Sie die abgespeicherte Stimme an ein oder mehrere Elemente in der Gruppe Sprach- und Soundschemata ankoppeln.

## Eine Stimme als Alias hinzufügen

Um einen neuen Stimmenalias hinzuzufügen, gehen Sie bitte folgendermaßen vor:

1. Öffnen Sie die Einstellungsverwaltung und geben Sie "Stimmenalias" im Sucheingabefeld ohne Anführungsstriche ein.
2. Drücken Sie **PFEIL RUNTER **, um zu Stimmenalias anpassen in den gefilterten Suchergebnissen der Strukturansicht zu gelangen und dann ** LEERTASTE **. Das Dialogfenster Stimmenalias wird geöffnet.
3. Aktivieren Sie den Schalter **Hinzufügen **.
4. Geben Sie einen Namen in das Feld Name des Alias ein. Es kann sich dabei um einen beliebigen Namen handeln, wir empfehlen jedoch logische Namen. Wenn die neue Stimme zum Beispiel Überschriften melden soll, könnten Sie den Namen "ÜberschriftenStimme" vergeben.
5. Mit **TAB ** springen Sie in die Ausklappliste Person. Diese Liste enthält alle verfügbaren Stimmen Ihrer Sprachausgabe, zusammen mit der Option Aktuelle Stimme. Wenn man eine beliebige Stimme außer der aktuellen auswählt, dann setzt JAWS jedes Mal, wenn dieser Alias aktiviert wird, diesen Sprecher ein. Wenn man Aktuelle Stimme auswählt, nimmt JAWS die notwendigen Änderungen der Geschwindigkeit und Stimmhöhe vor, verändert aber nicht die Stimme des Sprechers, wenn die Aktuelle Stimme als Alias verwendet wird. Beachten Sie, dass es sich bei Aktueller Stimme um die Stimme handelt, die JAWS verwendet, wenn der Alias aktiviert ist, nicht wenn der Alias erstellt wird.
6. Mit den nächsten beiden Schiebereglern stellen Sie die Stimmhöhe und die Geschwindigkeit für diesen Alias ein. Die Standardwerte sind die Werte, die JAWS zu der Zeit verwendet, wenn Sie den Alias erstellen. Sie können die Werte so lassen, wie sie sind, oder aber einen oder beide Werte so einstellen, dass der Alias schneller wiedererkannt wird.

## Aliase verändern

Um einen Stimmenalias zu verändern, markieren Sie den Stimmenalias zuerst in der Listenansicht. Als nächstes drücken Sie **ALT+Ä ** um den Schalter Ändern zu aktivieren. Der Dialog Alias ändern ist der gleiche wie der zum Hinzufügen neuer Aliase, außer dass sich die Werte für den Alias, den Sie verändern, auf die aktuellen Werte beziehen.

## Aliase entfernen

Um einen Stimmenalias zu entfernen, markieren Sie den entsprechenden Alias der Liste Stimmenalias. Drücken Sie **ALT+E **, um den Schalter ** Entfernen ** zu aktivieren.

## Gruppe Lesen

### Web Ausführlichkeitsstufe

Nutzen Sie diese Ausklappliste, um eine Ausführlichkeitsstufe für den virtuellen Cursor festzulegen. Die Ausführlichkeitsstufe legt den Umfang der von JAWS gesprochenen Informationen fest, wenn der virtuelle Cursor aktiv ist. Folgende Auswahlmöglichkeiten stehen zur Verfügung:

- **Hoch ** - JAWS spricht die meisten Informationen über Elemente auf einer Seite oder innerhalb eines Dokuments. Alles mit Ausnahme von Beginn und Ende der Anwendungs- und Dokumentregion wird in dieser Stufe angesagt.
- **Mittel ** - Diese Stufe ist konfiguriert, um die besten Benutzererfahrungen auf den meisten Seiten und Dokumenten zu bieten und ist die Standard-Ausführlichkeitsstufe. Standardmäßig wird JAWS Beginn und Ende von Flash Objekten oder Rahmen nicht anzeigen und nur den Beginn und das Ende von Navigations- und Hauptregionen ansagen. Beachten Sie, dass Name und Typ noch immer für alle Regionen angesagt wird, wenn die Schnellnavigationstasten ** R ** und ** UMSCHALT+R ** gedrückt werden, um Regionsweise zu springen, oder ** O ** und ** UMSCHALT+O **, um Artikelweise zu springen.
- **Niedrig ** - JAWS spricht so wenig strukturelle Informationen wie möglich über Elemente auf einer Seite oder innerhalb eines Dokuments. Standardmäßig wird in dieser Stufe nur der Beginn und das Ende von Tabellen und Gittern angesagt.

### Web Ausführlichkeitsstufen anpassen

Diese Reihe von Schaltern lässt Sie die Ausführlichkeitseinstellungen des virtuellen Cursors der Stufen Hoch, Mittel und Niedrig anpassen. Jede Ausführlichkeitsstufe enthält eine Voreinstellung. Wählen Sie eine der drei Stufen, und schalten Sie die Einstellungen an oder aus. Hierdurch wird ermöglicht, dass Sie mit einer Grundkonfiguration starten und diese nach Belieben anpassen können.

**Hinweis:** Wenn Sie die Ansage der Regionen deaktiviert haben, dann hören Sie weiterhin den Namen und den Typ einer Region, wenn Sie die Schnellnavigationstasten ** R ** und ** UMSCHALT+R ** verwenden. Ist JAWS so konfiguriert, dass Start und Ende eines Formularfleds angesagt werden, dann wird auch diese Information nicht gesprochen, wenn das Formularfeld nicht beschriftet ist und das Drücken von ** R ** oder ** UMSCHALT+R ** springt nicht in unbeschriftete Formularregionen.

### Leerzeilen beim Navigieren überspringen

Wenn aktiviert, dann überspringen JAWS und Fusion leere Zeilen, wenn mit dem virtuellen Cursor navigiert wird. Dieses verbessert das Lesen beim Navigieren auf Webseiten, in PDF Dokumenten, Hilfedateien und im JAWS Ergebnisbetrachter, wenn leere Zeilen nicht angesagt werden. Deaktivieren Sie dieses Kontrollfeld, wenn Sie beim Navigieren von Text auch leere Zeilen hören möchten.

Dieses Kontrollfeld ist standardmäßig aktiviert.

### Dokument und Webseiten beim Laden automatische lesen

Wenn aktiviert, dann startet JAWS das Lesen von Webseiten und PDF Dokumenten automatisch, sobald diese geladen sind. Deaktivieren Sie dieses Kontrollfeld, wenn Sie nicht möchten, dass JAWS den Lesevorgang automatisch startet. Das Deaktivieren dieser Einstellung ist speziell dann sinnvoll, wenn Sie JAWS in Verbindung mit einer Bildschirmvergrößerungssoftware verwenden, wo der Fokus unerwartet nach oben wegspringen würde, während Sie die Maus zum Navigieren durch das Dokument verwenden. Dieses Kontrollfeld ist standardmäßig aktiviert, wenn nur JAWS alleine läuft und ist standardmäßig deaktiviert, wenn Fusion oder FSReader verwendet wird.

**Hinweis:** Wenn Sie festlegen möchten, ob Nachrichten in Outlook beim Öffnen automatisch gelesen werden sollen, drücken Sie in Outlook die Tastenkombination ** EINFÜGEN+V **, um die Schnelleinstellungen zu öffnen, suchen Sie nach ** Nachrichten automatisch lesen ** und aktivieren oder deaktivieren Sie dieses Kontrollfeld mit der ** LEERTASTE **.

### Flexible Web aktivieren

Nutzen Sie diese Einstellung, um das Flexible Web ein- oder auszuschalten. Sie können auf das Flexible Web in Edge, Chrome oder Firefox durch Drücken von **EINFÜGEN+LEERTASTE, X ** zugreifen. Dieses Kontrollfeld ist standardmäßig aktiviert. Diese Einstellung steht nur in den JAWS Standardeinstellungen zur Verfügung (** STRG+UMSCHALT+D **).

### Sich wiederholenden Text auf neuen Seiten überspringen

Ist dieses Kontrollfeld aktiviert, überspringt JAWS sich wiederholenden Text, der oft auf einer Website zu finden ist. Der virtuelle PC Cursor wird am Beginn des neuen Textes auf jeder Seite platziert. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Zugriffstasten in Webseiten sprechen

JAWS sagt die Zugriffstasten von HTML Elementen an, indem es das Attribut AccessKey verwendet. Mit Zugriffstasten springt man direkt auf ein Element wie zum Beispiel einen Link oder ein Formularfeld. Deaktivieren Sie dieses Kontrollfeld, wenn JAWS keine Zugriffstasten auf Webseiten sprechen soll. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Zusammenfassung der benutzerdefinierten Seitenlesezeichen

Nutzen Sie diese Funktion, um automatisch alle Lesezeichen auf einer HTML Seite und den dazugehörigen Text aufzulisten. Wenn JAWS keine Zusammenfassung der benutzerdefinierten Seite anzeigen soll, wählen Sie Aus. Wählen Sie Nur sprechen, wenn JAWS sowohl den Namen jedes einzelnen Lesezeichens auf der aktuellen Seite als auch den Text, der sich in der Nähe des Lesezeichens befindet, lesen soll. Wenn Sie Informationen virtualisieren auswählen, zeigt JAWS die Informationen über die Lesezeichen der Seite im virtuellen Betrachter an. Im virtuellen Betrachter befindet sich ein Link mit dem Namen des Lesezeichens und der Text, der sich in der Nähe des Lesezeichens befindet.

## Gruppe Navigation

### Schlauer Navigationsmodus

Die schlaue Navigation bietet eine effizientere Möglichkeit, in webbasierten Anwendungen, Tabellen und Formularen zu navigieren. Die folgenden Optionen stehen zur Verfügung:

- **Steuerelemente:** Steuerelemente, inklusive (aber nicht nur beschränkt auf) Schalter, Links, Eingabefelder, Listenfelder, Strukturansichten und Überschriften, werden als einzelne Einheiten gewertet, wenn Sie zeichen- oder wortweise navigieren, so, wie man sich auch in einer Menüzeile mit ** PFEIL RECHTS ** und ** PFEIL LINKS ** bewegt. Zweimaliges schnelles Aufrufen von Zeichen Lesen, Wort Lesen und Zeile Lesen während ein Element fokussiert ist, wechselt zur zeichen- und wortweisen Navigation, so dass Sie die traditionelle Navigation per Zeichen, Wort, Zeile, Satz und Absatz nutzen können, um das Element genauer zu untersuchen, wie etwa der Text auf einem Schalter oder einem Link. Die schlaue Navigation per Steuerelement wird automatisch fortgesetzt, sobald Sie zu einem anderen Steuerelement durch Drücken von ** TAB **, ** PFEIL RAUF ** oder ** PFEIL RUNTER ** springen. Treffen Sie auf eine Tabelle, dann können Sie weiterhin die Tabellen-Lesebefehle (** STRG+ALT+PFEILTASTEN **) oder die Tabellenebene (** EINFÜGEN+LEERTASTE, T **) zum Navigieren in Tabellen verwenden.
- **Steuerelemente und Tabellen:** Wenn sich der Fokus in einer Tabelle befindet, dann bewegen die Tasten ** PFEIL RECHTS ** und ** PFEIL LINKS ** zwischen Spalten, während die Tasten ** PFEIL RAUF ** und ** PFEIL RUNTER ** zwischen Reihen bewegen. Wenn eine Zelle fokussiert ist, die Steuerelemente, wie etwa Überschriften oder Eingabefelder enthält, dann schalten Sie die Navigation für Steuerelemente durch zweimaliges schnelles Drücken der Befehle für Zeichen lesen, Wort lesen oder Zeile lesen ein, ein weiteres zweimaliges schnelles Drücken dieser Befehle wechselt zur zeichen- und wortweisen Navigation. Wenn die Zelle keine Steuerelemente enthält, dann können Sie nur zur zeichen- und wortweisen Navigation wechseln. Bewegen Sie den Fokus mit ** PFEIL LINKS ** oder ** PFEIL RECHTS ** von der aktuellen Zelle weg, dann wird zur vorherigen Navigationsebene umgeschaltet, drücken Sie also beispielsweise ** PFEIL RECHTS **, während Sie sich in der zeichen- und wortweisen Navigation befinden, dann gelangen Sie zurück zur schlauen Navigation in Tabellen, sobald Sie sich zu einer anderen Zelle bewegen. Wenn Sie sich nicht in einer Tabelle befinden, dann ist die Navigation so, wie bei der Einstellung für Steuerelemente.
- **Tabellen:** Bietet die gleiche Erfahrung wie die Einstellung Steuerelemente und Tabellen beim Navigieren innerhalb von Tabellen. Wenn Sie sich nicht in einer Tabelle befinden, ist die schlaue Navigation deaktiviert, und die Navigation nach Zeichen oder Wörtern funktioniert wie gewohnt.
- **Aus:** Die zeichen- oder wortweise Navigation in virtuellen Dokumenten funktioniert so, wie es schon immer in JAWS war. Dies ist die Standardeinstellung.

Wenn Sie die schlaue Navigation verwenden, wird JAWS immer im Bildschirmlayout arbeiten. Dadurch werden die Seiten im virtuellen Speicher so dargestellt, wie sie auch visuell auf dem Bildschirm zu sehen sind. Symbole der Werkzeugleisten oder andere Auflistungen von Steuerelementen, die visuell waagerecht auf dem Bildschirm angezeigt werden, werden im virtuellen Dokument in einer Zeile dargestellt, wodurch mit einem einzelnen Druck auf **PFEIL RUNTER ** die gesamte Werkzeugleiste übersprungen werden kann, anstatt dass man von Schalter zu Schalter springen muss, um hinter diese Leiste zu gelangen.

Um die schlaue Navigation temporär ein- oder auszuschalten, drücken Sie **EINFÜGEN+X **. Hierdurch wird auch der Dokumentpräsentationsmodus entsprechend angepasst. Wird ** EINFÜGEN+X ** gedrückt, um die schlaue Navigation zu aktivieren, dann wechselt JAWS automatisch zum Bildschirmlayout. Wird die schlaue Navigation ausgeschaltet, dann wechselt JAWS zurück zum einfachen Layout. Wenn Sie mit ** EINFÜGEN+X ** die schlaue Navigation temporär umschalten, dann schaltet es zurück auf die vorherige Einstellung, wemm Sie einen Befehl ausführen, der den Fokus vom aktuellen Browserfenster wegbewegt, wie etwa ** ALT+TAB **.

### Der Modus Dokumentpräsentation

Im Modus Dokumentpräsentation können Sie JAWS anweisen, wie HTML Seiten und andere virtuelle Dokumente (wie zum Beispiel PDF Dokumente) im virtuellen Buffer strukturiert werden sollen. Wenn Sie "Einfaches Layout" wählen, dann werden Links, Formularfelder, Schalter und andere Seitenelemente auf einer separaten Zeile angezeigt. Dieses ist das Standardlayout, wenn die schlaue Navigation nicht aktiviert ist. Wählen Sie "Bildschirmlayout", dann werden Seiten im virtuellen Buffer ähnlich wie auf dem Bildschirm sichtbar abgebildet. Wenn Seitenelemente visuell in der gleichen Zeile erscheinen, dann erscheinen Sie auch im virtuellen Speicher in der gleichen Zeile. Zusätzlich werden alle Elemente in einer einzelnen Tabellenreihe in der gleichen Zeile dargestellt, und die Spalten werden durch vertikale Balken getrennt. Wenn Sie sich zeilenweise nach oben oder nach unten bewegen, dann bleibt der Cursor in der richtigen Spalte. Die aktuelle Reihen- und Spaltenposition wird auch in Braille angezeigt. Der Modus Bildschirmlayout vereinfacht das Verständnis für das gesamte Tabellenlayout und verdeutlicht die Beziehungen der Tabellenzellen in jeder Reihe. Das Bildschirmlayout wird verwendet, wenn die schlaue Navigation auf Steuerelemente oder auf Steuerelemente und Tabellen gesetzt ist und kann nicht geändert werden, solange die schlaue Navigation aktiviert ist.

Diese Funktion ist immer dann sinnvoll, wenn Sie Tabelleninformationen aus virtuellen Dokumenten kopieren und einfügen möchten. Da der gesamte Text der Reihe in einer Zeile erscheint, können Sie den gesamten Text einer jeden Zelle in einer Reihe mit **UMSCHALT+PFEIL RUNTER ** markieren. Sie können auch mit ** F8 ** die gesamte Tabelle markieren (sofern Sie sich am Tabellenanfang befinden). Wenn Sie die Reihe(n) oder die Tabellen in ein anderes Dokument kopieren und einfügen, dann wird der Text einer jeden Zelle durch einen vertikalen Balken getrennt, so dass sich die Lesbarkeit erhöht. Sie können auch mit der erweiterten Funktionalität des [FSClipboard](JFW.chm::/using_jaws/Using_the_Clipboard.htm#FSClipboard) nicht zusammenhängende Reihen oder Reihen aus verschiedenen Tabellen kopieren, und diese dann alle zusammen in ein anderes Dokument einfügen.

### Maximale Zeilenlänge

Diese Einstellung bestimmt, wie lang eine einzelne Zeile sein soll, wenn eine HTML Seite im Modus Dokumentpräsentation gelesen wird. Das erleichtert das Lesen von langen Tabellen, da der gesamte Inhalt jeder Reihe einer Tabelle in eine einzelne Zeile passt. Die erhöhte Zeilenlänge verhindert, dass JAWS Reihen in mehreren Zeilen wiedergibt. Sobald Sie die Dokumentpräsentation wieder verlassen, liest JAWS die Seite in der normalen maximalen Zeilenlänge. Die Standardzeilenlänge im Modus Dokumentpräsentation beträgt 400 Zeichen, was für die meisten Tabellenreihen ausreicht, um in eine Zeile zu passen.

### Länge des Textblocks

Wenn Sie **JAWS TASTE+EINGABE ** drücken, um zum nächsten Text zu springen, der kein Link ist, sucht JAWS nach einem Textblock, der nicht Teil eines Links ist und so viele oder mehr Zeichen aufweist, wie der Wert, der hier angegeben wird. Der Standardwert ist 25.

### Zeilen pro Seite

Diese Einstellung legt fest, um wie viele Zeilen der virtuelle Cursor sich bewegt, wenn Sie **SEITE RAUF ** oder ** SEITE RUNTER ** drücken. Der Standardwert ist 24.

### Maximale Zeilenlänge

Diese Einstellung legt fest, wie viele Zeichen der virtuelle Cursor als eine Zeile ansieht. Der Standardwert ist 150.

### Bildschirmverfolgung Virtueller Cursor

Ist dieses Kontrollfeld aktiviert, folgt der Bildschirm dem virtuellen Cursor. Ist es nicht aktiviert, kann der virtuelle PC Cursor immer noch auf alle Elemente des HTML Dokuments zugreifen, die Anzeige am Bildschirm folgt jedoch nicht nach, wenn sich der virtuelle Cursor durch das Dokument bewegt. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Umbruch bei Navigation

Ist dieses Element markiert, springen die Befehle der Schnelltasten-Navigation weiter, wenn sie den Anfang oder das Ende eines HTML Dokuments erreichen. Ist diese Option nicht ausgewählt, weist JAWS Sie darauf hin, dass keine weiteren Übereinstimmungen wie zum Beispiel Tabellen gefunden wurden. Dieses Element ist standardmäßig aktiviert.

## Gruppe Bilder/Grafiken

### Grafiken einbeziehen

Nutzen Sie diese Auswahlschalter, um festzulegen, wie Sie möchten, dass JAWS Grafiken innerhalb von HTML Dokumenten ansagt.

- Wählen Sie **Alle Grafiken **, damit JAWS alle Grafiken in einem Dokument ansagt. Hat eine Grafik eine Bezeichnung, dann wird die Bezeichnung gelesen. Hat sie keine Bezeichnung, dann weist JAWS dieser Grafik eine einzigartige Nummer zu und liest diese.
- Wählen Sie **Bezeichnete Grafiken **, damit JAWS nur bezeichnete Grafiken ansagt.
- Wenn Sie **Keine Grafiken ** wählen, dann wird JAWS keinerlei Grafiken im Dokument ansagen.

### Gruppe Grafik Optionen

Mit diesen Auswahlschaltern legen Sie fest, welche Informationen JAWS ansagen soll, wenn man auf Grafiken auf Webseiten trifft. JAWS sammelt die Informationen aus den HTML "IMG" Element-Attributen wie zum Beispiel Alt, Title oder Src. Legen Sie fest, nach welcher Information JAWS zuerst suchen soll, bestimmen Sie, dass die längste Information verwendet werden soll oder legen Sie eine eigene Suchreihenfolge fest, nach der JAWS bei der Suche nach Informationen vorgehen soll. Wenn keine andere Information gefunden wird, verwendet JAWS die Information aus dem Src Attribut, die den Namen der Grafikdatei beinhaltet.

- **Titel **: Wenn diese Option ausgewählt wird, sucht JAWS zuerst nach der Information, die vom HTML Title Attribut geliefert wird und liest sie. Wenn kein Title Attribut gefunden wird, sucht JAWS nach der Information Alt Attribut und liest sie. Der Title Text wird am Bildschirm nicht angezeigt.
- **Alt Text **: Wenn diese Option ausgewählt wird, sucht JAWS zuerst nach der Information, die vom HTML Alt Attribut geliefert wird und liest sie. Wenn kein Alt Attribut gefunden wird, sucht JAWS nach der Information Title Attribut und liest sie. Der Alt Text wird am Bildschirm nicht angezeigt. Dieses ist die Standardeinstellung.
- **Tooltipp **: Wenn diese Option ausgewählt wird, sucht JAWS nach "OnMouseOver quoted Text", Alt Text und dann nach Title Text. JAWS spricht die zuerst gefundene Information. "OnMouseOver" Attribute werden häufig dazu verwendet, Skriptfunktionen zu starten, um Tool Tipps anzuzeigen, die nützliche Informationen enthalten. "OnMouseOver quotierter Text" ist Text, der durch eine Funktion im "OnMouseOver" HTML Attribut geliefert wird, in Form von: OnMouseOver="myfunction('information der Grafik')". Wird im "OnMouseOver" Attribut auf komplexeren Skriptcode verwiesen, so wird diese Information ignoriert.
- **Längsten verwenden **: Die längste Zeichenfolge an Informationen wird für jede Grafik gesprochen.
- **Das erste in der Liste gefundene Attribut **: Mit dieser Option können Sie festlegen, in welcher Reihenfolge JAWS nach Grafik-Informationen suchen soll. Sobald diese Option ausgewählt ist, wird das Eingabefeld Suchreihenfolge für Grafiken festlegen aktiviert, in dem Sie eine eigene Suchreihenfolge eingeben können.

### Unbezeichnete Grafiklinks

Mithilfe dieser Auswahlschalter legen Sie fest, wie JAWS grafische Links melden soll, wenn keine Alt Tags oder andere Informationen zur Identifizierung vorhanden sind. Standardmäßig spricht JAWS die Quellinformationen der Abbildung, die den Dateinamen der Grafik beinhalten.

## Gruppe Links

### Aufeinander folgende doppelte Links filtern

Aktivieren Sie dieses Kontrollfeld, damit JAWS nur Textlinks ansagt, wenn zwei aufeinanderfolgende Links auf einer Webseite, in der Regel ein Grafik- und ein Textlink, auf die gleiche URL verweisen. Dieses Kontrollfeld ist standardmäßig nicht aktiviert.

Wenn aktiviert, dann meldet JAWS nur den Textlink, wenn Sie sich auf der Webseite mit den **Pfeiltasten ** weiterbewegen. Wenn Sie sich auf der Webseite mit ** TAB ** oder ** UMSCHALT+TAB ** weiterbewegen, dann liest JAWS beide Links. Bitte beachten Sie, dass beide Links in der Linkliste erscheinen, wenn Sie ** JAWS TASTE+F7 ** drücken.

### Linktyp ansagen

### Links dieser Seite erkennen

Ist dieses Kontrollfeld aktiviert, sagt JAWS Links, die auf andere Stellen derselben Seite verweisen, als "Link dieser Seite" an. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Gruppe Textlinkoptionen

Diese Auswahlschalter legen fest, welche Informationen JAWS spricht, wenn es auf einen Textlink stößt. Die folgenden Optionen stehen zur Verfügung:

- **Titel **: Wenn diese Option ausgewählt wird, sucht JAWS zuerst nach der Information, die vom HTML Title Attribut geliefert wird und liest sie. Dieser Text wird am Bildschirm nicht angezeigt. Wenn kein Title Text gefunden wird, liest JAWS den Bildschirmtext.
- **Bildschirmtext **: JAWS liest den Bildschirmtext. Dieses ist die Standardeinstellung.
- **Tooltipp **: Wenn diese Option ausgewählt wird, sucht JAWS nach "OnMouseOver quoted Text" und liest ihn. "OnMouseOver" Attribute werden häufig dazu verwendet, Skriptfunktionen zu starten, um Tool Tipps anzuzeigen, die nützliche Informationen enthalten. Wenn kein "OnMouseOver quotierter Text" gefunden wird, spricht JAWS den Bildschirmtext des Links. "OnMouseOver quotierter Text" ist Text, der durch eine Funktion im "OnMouseOver" HTML Attribut geliefert wird, in Form von: OnMouseOver="myfunction('information about link')". Wird im "OnMouseOver" Attribut auf komplexeren Skriptcode verwiesen, so wird diese Information ignoriert.
- **Längsten verwenden **: JAWS spricht die längste Zeichenfolge für jeden Link.
- **Das erste in der Liste gefundene Attribut **: Mit dieser Option können Sie festlegen, in welcher Reihenfolge JAWS nach Link-Informationen suchen soll. Sobald diese Option ausgewählt ist, wird das Eingabefeld ** Suchreihenfolge ** für Grafiken festlegen aktiviert, in dem Sie eine eigene Suchreihenfolge eingeben können.

- **Bezeichnete Image Map Links **: Grafische Links, die alternierenden Text im HTML Dokument besitzen, werden angesagt.
- **Keine Image Map Links **: Grafische Links werden nicht angesagt.

### Image Map Links erkennen

Oft werden Teile einer Grafik mit Koordinaten versehen und als Links deklariert. Diese Auswahlschalter legen fest, welche Image Map Links angesagt werden. Die folgenden Optionen stehen zur Verfügung:

- **Alle Image Map Links **: Alle Image Map Links werden angesagt. Dieses ist die Standardeinstellung.
- **Bezeichnete Image Map Links **: Image Map Links, die mit alternativem Text im HTML Dokument versehen sind, werden angesagt.
- **Keine Image Map Links **: Es werden keine Image Map Links angesagt.

## Gruppe Überschriften und Rahmen

### Überschriftennavigation mit Zifferntasten 1-6

Nutzen Sie die Auswahlschalter dieser Gruppe, um festzulegen, was während der Navigation durch Überschriften passiert, wenn über die Zahlenreihe durch Überschriften bestimmter Ebenen navigiert wird. Die folgenden Einstellungen stehen zur Verfügung:

- **Auf Abschnitt beschränken ** beschränkt die Navigation darauf, Überschriften mit gleicher oder niedrigerer Priorität zu finden. Wenn Sie beispielsweise nach einer Überschrift der Ebene 3 suchen, wenn die nächste Überschrift aber eine Überschrift der Ebene 2 ist, wird die Suche gestoppt, da eine Überschrift der Ebene 2 den Beginn eines neuen Abschnitts anzeigt, dessen Ebene eine höhere Priorität als die gesuchte Überschrift hat.
- Wenn Sie **Überschriften in einem neuen Abschnitt lesen **, wird die nächste Überschrift der ausgewählten Ebene gefunden. Wenn diese Überschrift jedoch unter einer anderen Überschrift mit höherer Priorität liegt, werden die Überschriften mit der unmittelbar höheren Priorität gelesen, sodass Sie wissen, dass die neue Überschrift zu einem anderen Abschnitt gehört.
- Wenn **Sound abspielen bei neuem Abschnitt ** ausgewählt ist, wird bei der Überschriftennavigation die nächste Überschrift der angegebenen Ebene gefunden. Wenn sie jedoch unter einer anderen Überschrift mit höherer Priorität liegt, wird ein Ton abgespielt, um darauf hinzuweisen, dass die Überschrift zu einem anderen Abschnitt gehört.

Standardmäßig können Sie sich mit der Zahlenreihe ohne Beschränkungen durch die Überschriftsebenen bewegen und JAWS sagt für einen neuen Abschnitt die Überschriftsebene an, sofern sie sich ändert.

### Überschriften ansagen

Mit diesen Auswahlschaltern legen Sie fest, wie JAWS Überschriften ansagen soll. Die Standardeinstellung lautet Überschriften mit Ebene ansagen. Wenn Überschriften ordnungsgemäß gesetzt wurden, zeigen sie die Beziehung jedes einzelnen Abschnittes der Seite zur Gesamtseite auf.

Wählen Sie eine der folgenden Optionen:

- **Keine Überschriften ** - JAWS meldet keine Überschriften.
- **Nur Überschriften ** - JAWS sagt "Überschrift", sobald Sie auf eine Überschrift springen, aber die Überschriftsebene wird von JAWS nicht gemeldet.
- **Überschriften mit Ebene ** - JAWS sagt "Überschrift", gefolgt von der Überschriftsebene, sobald Sie auf eine Überschrift treffen. Dies ist die Standardeinstellung.
- **Sound abspielen ** - JAWS spielt einen Sound ab, sobald Sie auf eine Überschrift springen. Sie können jeder Überschriftsebene einen anderen Sound zuweisen. Im Sprach- und Soundmanager wählen Sie die gewünschten Sounds aus.
- **Überschrift mit Stimme ansagen ** - JAWS verwendet beim Lesen von Überschriften eine andere Stimme. Sie können jeder Überschriftsebene eine andere Stimme zuweisen. Im Sprach- und Soundmanager wählen Sie die gewünschten Stimmen-Aliase aus.

### Eingebettete Rahmen ignorieren

Weist JAWS an, eingebettete Rahmen zu ignorieren, wenn Sie die darin enthaltenen Informationen nicht hören möchten. Eingebettete Rahmen werden oft dazu genutzt, Werbung und Annoncen anzuzeigen. Bedenken Sie aber, dass es auch Webseiten gibt, die wichtige Informationen in eingebetteten Rahmen anzeigen können. Standardmäßig liest JAWS den Inhalt von eingebetteten Rahmen.

## Die Gruppe Tabellen

### Spannweite von Spalten in Braille anzeigen

Aktivieren Sie dieses Kontrollfeld, dann fügt JAWS pro Spalte einen vertikalen Balken zwischen den Tabellenzellen ein. Dadurch wird das Lesen von HTML Tabellen vereinfacht, die Zellen enthalten, die sich über mehrere Spalten erstrecken. Wenn sich zum Beispiel eine Zelle über zwei Spalten erstreckt, dann werden hinter dem Inhalt dieser Zelle zwei vertikale Balken auf Ihrer Braillezeile angezeigt. Dadurch erhalten Sie einen besseren Eindruck über das optische Layout von HTML Tabellen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Tabellentyp erkennen

JAWS kann so eingestellt werden, dass nur Datentabellen oder alle Tabellen, auch Tabellen, die zur Seitenformatierung verwendet werden, erkannt werden. Die Verschachtelungsebene jeder Tabelle wird ebenso angesagt. Tabellen werden häufig dazu verwendet, die Formatierung für das Seitenlayout bereitzustellen, so dass die Einstellung Alle Tabellen dazu führen kann, dass auf einigen Seiten umfangreiche Informationen geliefert werden. Die Standardeinstellung lautet Nur Daten-Tabellen.

## Die Gruppe Blickfang

Nutzen Sie die Einstellungen dieser Gruppe, um festzulegen, wie Blickfang-Hervorhebungen beim Lesen von Webseiten, in HTML- und PDF-Dateien angesagt werden sollen.

### Blickfang-Hervorhebungen anzeigen

Nutzen Sie diese Option, um festzulegen, wie Blickfang Hervorhebungen angesagt werden. Die Textsegmente der Liste für Blickfang-Hervorhebungen zeichnen sich durch ihre visuelle Gestaltung aus. Die folgenden Optionen stehen zur Verfügung:

- **Beim Laden der Seite ** - JAWS sagt die Anzahl der Blickfang-Hervorhebungen auf einer Seite zusammen mit der Anzahl der Links, Überschriften, Regionen und Formularfelder an, nachdem die Seite vollständig geladen wurde. Dies ist die Standardeinstellung.
- **Beim Navigieren ** - JAWS weist auf die Hervorhebungen auf einer Seite hin, indem ein Ton abgespielt wird, wenn Sie zeichenweise, wortweise, zeilenweise, satzweise, absatzweise navigieren oder beim Alles lesen.
- **Beides ** - JAWS weist sowohl beim Laden der Seite, als auch beim Navigieren auf Blickfang-Hervorhebungen hin.
- **Aus ** - Schaltet die Hinweise auf Blickfänge aus.

### Empfindlichkeit

Nutzen Sie diese Option, um die Empfindlichkeit für die Bestimmung einer Blickfang-Hervorhebung zu erhöhen. Mit einer höheren Empfindlichkeit werden mehr Ergebnisse angezeigt, da auch kleinere Textabschnitte auf spezielle Textattribute untersucht werden.

### Elemente einbeziehen

Aktivieren oder deaktivieren Sie die Kontrollfelder dieser Gruppe, um auszuwählen, welche HTML-Elemente bei der Analyse einer Webseite nach Blickfang-Hervorhebungen berücksichtigt werden sollen. Standardmäßig sind nur erforderliche und ungültige Felder aktiviert.

## Die Gruppe Diverses

### Flash auf Webseiten ignorieren

Dieses Kontrollfeld ist standardmäßig nicht aktiviert, so dass Sie Flashanimationen lesen können, sobald diese auftauchen. Wenn Sie Flashanimationen lieber nicht lesen möchten, markieren Sie dieses Feld, damit diese von JAWS ignoriert werden.

### Bezeichnungen für Formularfelder sprechen

Mithilfe dieser Auswahlschalter legen Sie fest, wie JAWS Prompts für Formularfelder suchen soll. JAWS greift auf die Formularinformationen entweder über die Attribute Label, Title oder Alt zu. Mit dieser Einstellung teilen Sie JAWS mit, welches Attribut oder welche Kombination beim Sprechen von Promptinformationen für Formularfelder verwendet werden soll. Die Standardeinstellung lautet Label Tag favorisieren.

### Seitenaktualisierungsfilter

Webseiten sind manchmal so eingestellt, dass sie Informationen automatisch aktualisieren und den Bildschirm dadurch neu aufbauen. Dies kann zur Folge haben, dass JAWS immer wieder vom Seitenanfang zu lesen beginnt und das Lesen somit sehr schwer macht. Um festzulegen, wie häufig eine Seite den Inhalt aktualisiert, nutzen Sie die Option Aktualisierungsfilter für Seiten. Wählen Sie Bei Bedarf aktualisieren, um die Seite so häufig zu aktualisieren, wie die Komponenten auf der Seite diese neu laden. Dies ist die Standardeinstellung. Wählen Sie Alle 60 Sekunden aktualisieren, um die Seite in 60-Sekunden-Intervallen zu aktualisieren. Wählen Sie Alle Aktualisierungen unterdrücken, um die Seite niemals zu aktualisieren, solange sie fokussiert ist. Hinweis: Wenn Sie die Option Aktualisierungen unterdrücken gewählt haben, kann es passieren, dass einige Seiten nur unzureichend gelesen werden. Wir empfehlen, dass Sie die Aktualisierungen nur auf Seiten unterdrücken, auf die Sie ansonsten nur schwer zugreifen können. Öffnen Sie dazu in Ihrem Webbrowser die entsprechende Seite und nutzen Sie die Funktion Angepasste Web-Einstellungen.

### Link-Aktivierung

Diese Einstellung betrifft die Aktivierung von Links, wenn der Anwender in virtuellen Dokumenten die **Eingabetaste ** drückt.

Ist diese Einstellung auf "EINGABE simuliert Mausklick" gesetzt, dann bewirkt das Drücken der **Eingabetaste ** auf einem Link, dass ein linker Mausklick auf dem Link ausgeführt wird. Dieses ist das Verhalten von JAWS in früheren Versionen vor JAWS 15.

Ist diese Einstellung auf "EINGABE sendet Eingabetaste" gesetzt, dann bewirkt das Drücken der **Eingabetaste ** auf einem Link, dass die ** Eingabetaste ** an das System durchgereicht wird. Vielleicht überlegen Sie sich, diese Einstellung auf "EINGABE sendet Eingabetaste" zu setzen, wenn die ** Eingabetaste ** ein Webelement nicht aktiviert, solange Sie nicht vorher ** JAWS Taste+3 ** drücken, um die Funktion Taste durchreichen zu aktivieren.

Die Standardeinstellung ist EINGABE simuliert Mausklick.

### Optionen für Schalterbeschriftungen

Wählen Sie die Information, die JAWS sprechen soll, wenn sich Schalter in Formularen auf Webseiten befinden. JAWS kann Title, Bildschirmtext, Alt, Wert oder die längste Zeichenkette für die Information nutzen. Der Standardwert ist Bildschirmtext.

### Elementattribute sprechen

Wählen Sie dieses Kontrollfeld, damit JAWS Attribute von HTML Elementen auf Webseiten ansagt. Die von JAWS angesagten Attribute werden über das Sprach- und Soundschema auf der Seite Attribute festgelegt. Ein Deaktivieren dieses Kontrollfelds unterdrückt die Ansagen von allen Attributen, die im Sprach- und Soundschema aktiviert sind. Zum Unterdrücken oder Ändern solcher Ansagen wird daher empfohlen, das Verhalten für die gewünschten Attribute auf der Seite Attribute im Sprach- und Soundschema vorzunehmen. Dieses Kontrollfeld ist standardmäßig aktiviert.

### Reservierte Tasten für Web-Anwendungen erlauben

Einige interaktive Webseiten können Kurztasten zum Ausführen von herkömmlichen Aufgaben anbieten. Zum Beispiel können Sie auf der Facebook Seite **P ** drücken, um einen Status zu posten, ** L ** drücken, um eine Meldung zu liken, ** C ** drücken, um eine Meldung zu kommentieren, oder mit ** J ** und ** K ** zwischen den Neuigkeiten scrollen. Diese Tasten werden aber auch als JAWS Schnellnavigationstasten verwendet. Wenn dieses Kontrollfeld aktiviert ist, dann werden alle JAWS Kurztasten, die auch für die Webanwendung definiert sind, dazu genutzt, die spezielle Aufgabe auf der Seite auszuführen, anstatt des JAWS Befehls. Damit dieses korrekt funktioniert, muss die Seite auch eine Zeile mit Quellcode enthalten, der JAWS mitteilt, welche Kurztasten durch die Webseite definiert sind. Wenn diese Information nicht zur Verfügung steht, dann werden alle JAWS Befehle weiterhin wie gewohnt ausgeführt. Dieses Kontrollfeld ist standardmäßig aktiviert.

## Die Gruppe Fensterklassen

Eine Fensterklasse ist eine hierarchische Programmstruktur für Fenster oder Steuerelemente. Sind für die Nutzung von Eingabe- und Kontrollfeldern keine Standardklassenbezeichnungen genutzt worden, dann findet JAWS diese Bezeichnungen möglicherweise nicht. Nutzen Sie die Elemente im Dialogfenster Fensterklassen (wie nachfolgend beschrieben), um Fenstern und Steuerelementen Standardklassenbezeichnungen zuzuweisen, so dass JAWS auf diese leichter zugreifen kann. Die ist sehr hilfreich in Programmen, wo JAWS Steuerelemente, wie Eingabe- oder Kontrollfelder, nicht erkennt. Auch wenn das Steuerelement wie in anderen Programmen funktioniert, hat es wahrscheinlich einen anderen Namen. Indem Sie solchen Klassen die Standard-Klasse zuweisen, stellen Sie sicher, dass JAWS diese zuverlässig erkennt und ansagt.

Um diesen Dialog Fensterklassen zu öffnen, gehen Sie bitte folgendermaßen vor:

1. Öffnen Sie die Einstellungsverwaltung und geben Sie "Fensterklassen" im Sucheingabefeld ohne Anführungsstriche ein.
2. Drücken Sie **PFEIL RUNTER **, um zu Fensterklassen anpassen in den gefilterten Suchergebnissen der Strukturansicht zu gelangen und dann ** LEERTASTE **. Das Dialogfenster Fensterklassen wird geöffnet.

## Neue Klasse

Dieses Eingabefeld enthält den Fensterklassennamen des aktuellen Fensters oder Steuerelements, welches reklassiert werden kann.

## Zuweisen an

Dieses Listenfeld enthält die Fensterklassen, die JAWS bekannt sind. Wählen Sie diejenige Klasse aus, die Sie dem Klassennamen im Eingabefeld Neue Klasse zuweisen möchten.

## Zugeordnete Klassen

Dieses Listenfeld enthält diejenigen unbekannten Fensterklassen, die bereits bekannten Fensterklassen zugewiesen wurden. Dies bezieht sich auf die JCF-Datei, die gerade geöffnet ist.
